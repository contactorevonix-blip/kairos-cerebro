🔄 Story Development Cycle (SDC) — Primary Workflow

Complete 5-phase flow for all development: create → validate → implement → qa-gate → push

Phase 1: Create (@sm)

Task: create-next-story.md | Input: PRD, epic context | Output: story.md (Draft)

📝 @sm *draft
→
{epicNum}.{storyNum}.story.md

Phase 2: Validate (@po)

Task: validate-next-story.md | Decision: GO (≥7/10) → Ready | NO-GO → fixes

✅ @po *validate
→
10-point checklist

Phase 3: Implement (@dev)

Task: dev-develop-story.md | Modes: YOLO | Interactive | Pre-Flight | CodeRabbit: max 2 iter

💻 @dev *develop
→
Ready → InProgress → InReview

Phase 4: QA Gate (@qa)

Task: qa-gate.md | 7 quality checks | Verdicts: PASS/CONCERNS → Done | FAIL → @dev

🧪 @qa *qa-gate
→
PASS/CONCERNS/FAIL/WAIVED

Phase 5: Push (@devops)

Authority: Only after QA PASS/CONCERNS/WAIVED

⚡ @devops *push
→
git push + gh pr create
