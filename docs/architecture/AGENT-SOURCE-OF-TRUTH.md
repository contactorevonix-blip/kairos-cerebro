## 4. Divergence Classification (AC2)

### Intentional — per-IDE format

The `Synced` counts already account for format differences (`*.agent.md` for GitHub Copilot, `*.mdc` for Cursor, `SKILL.md` subdirs for Kimi). These are produced by the source via each transformer — **not drift**.

### Intentional — not-yet-generated targets (`Missing`)

`gemini`, `gemini-commands`, `cursor`, `kimi` show `Missing` because their folders have **never been generated** for this repo. Running `ide-sync sync --ide <name>` would create them from the same source. This is expected, not accidental drift.

### Intentional — `Orphaned` files

| Orphaned set | Target(s) | Classification |
|--------------|-----------|----------------|
| 46 squad/expert agents (`aiox-cerebro`, `forge-*`, `kahneman`, `squad-chief`, …) | `.claude/commands`, `.claude/skills` | **Intentional** — these are **squad/project agents**, a *different* agent system not sourced from `.aiox-core/development/agents`. They legitimately live only in `.claude`. |
| 4 redirect stubs (`aiox-developer`, `aiox-orchestrator`, `db-sage`, `github-devops`) | `.codex`, `.antigravity` | **Intentional** — deprecated-agent **redirect stubs** generated by the ideSync redirect generator (e.g. `db-sage → @data-engineer`). Benign; tolerated by `failOnOrphaned: false`. |

> **`redirects: {}` decision:** the redirect map in `core-config.yaml` is intentionally empty. A *global* redirect map would create **false drift** in `.claude/commands` (where `db-sage.md` is a real squad agent, not a redirect) and **false "missing"** in github/gemini/cursor/kimi (which never received stubs). With an empty map, drift is 0 everywhere and the 4 codex/antigravity stubs remain as benign orphans.

### Accidental — found and fixed

| Accidental drift | Root cause | Fix |
|------------------|------------|-----|
| **`core-config.yaml` unparseable** | A parent-less stale list of protected paths (`- .aiox-core/infrastructure/**` …) plus an `exceptions:` key was left dangling under `autoClaude.qa`, a remnant of a relocated `boundary` block. js-yaml failed with *"bad indentation of a mapping entry (373:5)"*, so `loadConfig` silently fell back to built-in defaults (which carry a non-empty redirects map), corrupting every drift signal. | Removed the dead lines; the active `boundary:` config (top-level, line ~168) is unaffected. The drift report above is the result **after** this fix. |

No agent **content** required re-sync — once the config parsed correctly, drift was already 0 across all generated targets.

## 5. failOnDrift Smoke Test (AC5)

Verify-only proof that the `failOnDrift` / `--strict` mechanism actually fires:

```
# Baseline (no drift)
$ node .aiox-core/infrastructure/scripts/ide-sync/index.js validate --ide codex --strict
  → exit code 0

# Inject drift into a synced target file
$ printf '\n<!-- SIMULATED DRIFT -->\n' >> .codex/agents/dev.md
$ node .aiox-core/infrastructure/scripts/ide-sync/index.js validate --ide codex --strict
  → "| Drift | 1 |"  +  "Validation failed in strict mode"  +  exit code 1

# Restore
$ git checkout .codex/agents/dev.md   # (or restore from backup)
$ node ... validate --ide codex --strict
  → exit code 0
```

**Result:** drift present → **exit code 1** (gate fires); drift absent → **exit code 0**. The mechanism is proven live.

## 7. How to Run the Drift Audit

```bash
# Full validation report (all targets)
node .aiox-core/infrastructure/scripts/ide-sync/index.js validate --verbose

# CI / gate mode (exit 1 on drift)
node .aiox-core/infrastructure/scripts/ide-sync/index.js validate --strict

# Single target
node .aiox-core/infrastructure/scripts/ide-sync/index.js validate --ide codex --strict

# Re-sync from source (fixes accidental drift the canonical way — never edit targets by hand)
node .aiox-core/infrastructure/scripts/ide-sync/index.js sync

# Automated test backstop (this story's AC1/AC2/AC5/AC6)
node --test tests/agents/agent-drift-audit.test.js
```

> The pipeline's runtime deps (`js-yaml`, `fs-extra`) resolve from `.aiox-core/node_modules`.
