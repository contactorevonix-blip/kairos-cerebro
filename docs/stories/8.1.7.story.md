# Story 8.1.7: Metrics Persistence

**Status:** Ready
**Story ID:** 8.1.7
**Created:** 2026-06-11
**Epic:** EPIC-8 — Framework Evolution (Phase 1: Observability & Metrics)
**PRD Source:** `docs/prd/epic-8/phase-1-observability.md#817-metrics-persistence-15sp`
**Story Points:** 1.5

---

## Story

**As a** framework maintainer,
**I want** a PostgreSQL schema with migrations, retention policy, and seed/backup processes for metrics storage,
**so that** historical metrics are durably persisted, queryable, and bounded in size over time.

---

## Acceptance Criteria

1. Database schema: `migrations/001-metrics-init.sql`
   - Tables: `metrics` (id, key, value, timestamp, source), `metric_history` (aggregated)
   - Indexes: timestamp, article, agent (fast range queries)
   - Retention: Keep 1y data (auto-purge older than 365 days)
2. Migration system: Flyway or node-migrate compatible
3. Seed data: Load 30 days of existing logs into DB
4. Backup strategy: DB exports weekly to `backups/metrics-YYYY-MM-DD.sql`

**Testing:**
- Persist 1000 metrics, verify query <500ms
- Auto-purge oldest records, verify count stays <500k
