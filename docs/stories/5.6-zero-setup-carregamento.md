# Story 5.6 — Carregamento Automático / Zero Setup

**Epic:** AIOX-OPS (AIOX Operacional 100%) | **Story ID:** 5.6 | **Effort:** 5sp | **Status:** Ready | **Executor:** @dev | **Quality Gate:** @qa

## Story

**As a** developer cloning KAIROS_CEREBRO for the first time,
**I want** `settings.json` and `core-config.yaml` to load all automation without any manual setup steps,
**so that** the system is fully operational immediately after `git clone` — zero friction.

## Goal

Garantir que `settings.json` + `core-config.yaml` carregam toda a automação sem passos manuais. Validar SessionStart hook, hooks async, e que um "fresh clone simulation" do repo fica operacional sem setup adicional. Identificar e documentar qualquer "manual setup residual" que ainda exista.

## Contexto / Problema

EPIC-AIOX-OPERACIONAL.md §1 define a visão: "zero setup manual". Actualmente, sessões anteriores do STATE.md mencionam passos de configuração que foram feitos manualmente. Esta story verifica que a promise "zero setup" é verdadeira — ou documenta os passos manuais que ainda são necessários e cria um plano para os eliminar.

AUDIT-REPORT.md D2.4 confirmou que `hook-metrics.json` está sincronizado (2026-06-08) e `layersLoaded: 2`, mas a sequência de carregamento completa nunca foi validada numa simulação de fresh clone.

## Acceptance Criteria

- [ ] **AC1:** Checklist de carregamento automático completada — para cada componente de automação, verificar que carrega sem intervenção:
  - [ ] `session-start.cjs` hook dispara no início de sessão
  - [ ] `synapse-engine.cjs` carrega layers conforme bracket
  - [ ] `agent-activation-tracker.cjs` regista agente activo
  - [ ] `enforce-*.cjs` hooks activos para Art. II-VII
  - *Given* nova sessão Claude Code, *When* a sessão inicia, *Then* `hook-metrics.json` mostra `layersLoaded > 0` e `gatesEnforced` (ou zeros documentados como esperados).

- [ ] **AC2:** `settings.json` validado — todos os hooks registados têm: ficheiro existente (AC1 da Story 5.1 confirma), timeout razoável (< 30s), e async flag correcta.
  - *Given* `settings.json`, *When* inspeccionado, *Then* 0 entradas com `file_exists: false` que sejam críticas (enforcement hooks).

- [ ] **AC3:** `core-config.yaml` validado — campos obrigatórios presentes: `devStoryLocation`, `prd.*`, `boundary.frameworkProtection`, e todos os campos documentados em CLAUDE.md como necessários.
  - *Given* `core-config.yaml`, *When* lido, *Then* todos os campos do contrato público (CLAUDE.md/AGENTS.md) estão preenchidos.

- [ ] **AC4:** Fresh clone simulation executada — simular os passos que um developer faria ao fazer `git clone` e abrir o Claude Code. Documentar: (1) passos automáticos que funcionam, (2) passos manuais que ainda são necessários.
  - *Given* documentação do fresh clone simulation, *When* seguida passo a passo, *Then* a lista de passos manuais é <= 1 (ex: apenas configurar API keys de MCP).

- [ ] **AC5:** Qualquer "manual setup residual" documentado em `docs/SETUP-RESIDUAL.md` com: o que ainda é manual, porquê não pode ser automatizado (ou pode?), e story de resolução proposta (se aplicável).

- [ ] **AC6:** Output da Story 5.1 (Hook Automation Audit) consultado — hooks com FAIL na auditoria impactam o zero-setup? Documentar no relatório.

## Scope

**IN:**
- Validação de `settings.json` e `core-config.yaml` para completude e correctness
- Checklist de carregamento automático (todos os hooks críticos)
- Fresh clone simulation (documentar passos)
- Relatório de manual setup residual

**OUT:**
- Implementar automação nova para eliminar setup manual (isso é outra story se necessário)
- Modificar `settings.json` core (apenas validação; mudanças de settings são scope de outras stories)
- Modificar L1/L2
- Configurar MCP servers (autoridade exclusiva de @devops)

## Dependencies

- **Prerequisito:** Story 5.1 (Hook Automation Audit) — output confirma quais hooks funcionam, informando a checklist de carregamento.
- **Bloqueia:** 5.7 (consolida outputs — inclui estado de zero-setup).
- **Recursos:** `.claude/settings.json`, `.aiox-core/core-config.yaml`, `.synapse/metrics/hook-metrics.json`, CLAUDE.md, AGENTS.md.

## Complexity Estimate

5sp — validação de configuração + fresh clone simulation + relatório de residual. Complexidade média: sem lógica nova, mas requer atenção a detalhes de carregamento e validação de campos de configuração.

## Business Value

Concretiza a promise "zero setup manual" do epic. Um developer que clone o repo pela primeira vez tem um sistema operacional imediatamente — não há fricção de configuração. Elimina uma fonte de confusão recorrente ("porque é que os hooks não carregaram?").

## Risks

- **R1 (MEDIUM):** Fresh clone simulation pode revelar dependências de sistema (PATH, Node version, etc.) que não são automáticas. Mitigação: documentar como SETUP-RESIDUAL, não falhar a story.
- **R2 (LOW):** `core-config.yaml` pode ter campos obsoletos ou em conflito. Mitigação: reportar no relatório, não remover (backward-compatible).

## Dev Notes

### Componentes de carregamento automático (AC1)

Ordem de carregamento esperada (fonte: `.synapse/metrics/hook-metrics.json` + AUDIT-REPORT.md D2.4):
1. `session-start.cjs` → regista início de sessão
2. `synapse-engine.cjs` → carrega layers (constitution + global = 2 layers)
3. `agent-activation-tracker.cjs` → regista agente activo
4. `pre-tool-use-validator.cjs` → valida inputs antes de tool calls
5. `enforce-*.cjs` → gates de enforcement (Art. II-VII)

`hook-metrics.json` actual mostra: `layersLoaded: 2`, `layersSkipped: 6`, `totalRules: 73` — usar como baseline.

### Campos obrigatórios em core-config.yaml (AC3)

Campos mencionados em CLAUDE.md como necessários:
- `devStoryLocation` (localização das stories)
- `prd.*` (configuração do PRD)
- `architecture.*` (configuração de arquitectura)
- `workflow.*` (configuração de workflows)
- `boundary.frameworkProtection` (toggle L1-L4)
- `coderabbit_integration.enabled` (confirmado: true)
- `autoClaude.*` (flags de automation — scope de Story 5.3)

### Fresh clone simulation (AC4)

Passos esperados de um fresh clone:
1. `git clone <repo>`
2. `npm install`
3. Claude Code abre → session-start.cjs dispara?
4. Hooks carregam automaticamente via settings.json?
5. SYNAPSE carrega layers?

Documentar resultado real em `docs/SETUP-RESIDUAL.md`.

### Testing

- **Abordagem:** Leitura e validação de `settings.json` e `core-config.yaml` via scripts de verificação
- **Output:** Checklist completada + `docs/SETUP-RESIDUAL.md`
- **Validação:** `npm test` deve continuar a passar
