## Details

### Testing Approach

**MAESTRO Layer Validation:**
- **L1 (Core):** Schema design patterns, migration framework
- **L2 (Templates):** Migration templates, RLS policy templates
- **L3 (Config):** Database connection config (externalized)
- **L4 (Runtime):** @data-engineer operates on project schemas

**Task-First Contract:**
```
Task: Test @data-engineer Agent
  Inputs: [architecture decision, data model]
  Outputs: [DDL, migration script, RLS policies]
  Success: All 8 ACs passing, schema production-ready
```

### Database Design Workflow

@data-engineer follows schema design task:
1. **Architect Decision:** Receive design authority from @architect
2. **Data Model:** Design entities, relationships, constraints
3. **DDL Generation:** Create CREATE TABLE/INDEX statements
4. **RLS Policies:** Implement row-level security
5. **Migration Script:** Generate safe migration (reversible)
6. **Deployment:** Execute migration (test environment first)

### Research Alignment

**Clean Architecture** (research rec #1):
- Database is peripheral layer (not central business logic)
- @data-engineer owns implementation, @architect owns design

**Production Readiness** (research rec #5):
- Migrations reversible (backup, rollback capability)
- RLS policies documented and audited
