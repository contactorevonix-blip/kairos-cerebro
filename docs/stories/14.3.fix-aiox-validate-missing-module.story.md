# Story 14.3: Fix `aiox validate` Missing Module Error

**Status:** Draft

**Owner:** @sm (creation) / @dev (implementation)

**Track:** Quick Flow

---

## Story Statement

**As a** user of the AIOX CLI,  
**I want** the `aiox validate` command to run successfully,  
**so that** I can verify AIOX installation integrity without errors.

---

## Problem

`node bin/aiox.js validate` fails with:
```
Error: Validator module failed to load: Cannot find module 'C:\...\KAIROS_CEREBRO\packages\installer\src\installer\post-install-validator'
```

The CLI tries to load `post-install-validator` from a path that doesn't exist. This blocks installation verification.

---

## Acceptance Criteria

- [ ] **AC1:** Identify root cause: is the module missing, or is the path wrong?
      - Check if `post-install-validator.js` exists anywhere in the project (Glob search)
      - Check if `.aiox-core/cli/commands/validate/index.js` references a non-existent path
      
- [ ] **AC2:** Fix the issue:
      - Option A: Create missing `post-install-validator.js` (if spec is known)
      - Option B: Fix the require path in validate/index.js (if module exists but path is wrong)
      - Option C: Comment out validator if it's non-critical and not yet implemented
      
- [ ] **AC3:** Verify: `node bin/aiox.js validate` runs without error (output may be "validation incomplete" if module is stubbed, but no crashes)

- [ ] **AC4:** No impact on other CLI commands

---

## Dev Notes for @dev Implementation

### Investigation Checklist
1. Grep for `post-install-validator` in the codebase (all occurrences)
2. Check if `packages/installer/src/installer/` directory exists
3. Check `.aiox-core/cli/commands/validate/index.js` line ~X for the require statement
4. Determine if this is a vendored file or project-local file

### Likely Fix
- **If module path is wrong:** update require path in validate/index.js
- **If module is missing:** create a minimal stub that returns validation "not yet implemented" rather than crashing
- **If deleted intentionally:** remove the require statement and stub the validate function

### Quick Testing
After fix:
```bash
node bin/aiox.js validate
```
Should output success message or "validation incomplete" — not an error.
