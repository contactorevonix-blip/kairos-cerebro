# Story 9.4: Art. IV No-Invention Enforcement Gate

**Story ID:** 9.4

## Description

Implement Constitutional Article IV (No Invention) enforcement gate. Block spec.md changes that introduce untraced statements (features not present in requirements/research/ADRs).

**Article IV Statement:** Every statement in spec.md must trace to: FR-*, NFR-*, CON-*, or research finding. No invented features.

## Scope

**IN:**
- Detect Write/Edit operations on *spec.md files
- Parse spec content for untraced statements
- Validate traceability tags: [FR-X.Y], [NFR-A.B], [CON-*], [Research: source]
- Block untraced statements
- Permissive mode via env var
- 3+ test cases

**OUT:**
- Spec.md authoring (that's @pm @architect)
- Requirement ID validation (format checking only)
- Research source verification (assumes source is valid)
- Historical spec audit (current commits only)

## Acceptance Criteria

- [ ] Gate detects Write/Edit operations on files matching `*spec.md`
- [ ] Parse spec content for statements (lines starting with "-" or "•")
- [ ] Each statement must have traceability tag: `[FR-X.Y]`, `[NFR-A.B]`, `[CON-*]`, or `[Research: source]`
- [ ] If statement missing tag → BLOCK (decision: `block`)
- [ ] Permissive mode: Set `AIOX_NO_INVENTION_PERMISSIVE=1` to downgrade to `warn` (not BLOCK)
- [ ] Decision logged: article=`art-iv-no-invention`, reason shows untraced statement
- [ ] Test cases:
  - [ ] Add spec statement with `[FR-1.2]` → ALLOWED
  - [ ] Add spec statement without tag → BLOCKED
  - [ ] Permissive mode + untraced → WARN logged
- [ ] All 4 pre-commit gates PASS

## Risks Identified

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| False positive: legitimate multi-line statement parsed as separate | Medium | Medium | Line-based parsing; test with complex specs |
| Tag format too restrictive | Low | Medium | Regex allows flexible spacing; document tag format |
| Permissive mode abused | Low | Low | Audit trail; permissive mode logs all bypasses |
| Performance on large spec.md | Very Low | Low | Parse only changed lines (diff-based), not full file |
