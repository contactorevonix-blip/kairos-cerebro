## Story

**As a** framework reliability engineer,
**I want** constitutional gates to automatically retry on transient failures (timeouts, network errors),
**so that** temporary glitches don't block development (only real failures stop work).

---

## Acceptance Criteria

- [x] **AC1:** Module `.aiox-core/core/gates/gate-retry.js` created
- [x] **AC2:** Failure classification:
  - Transient (timeout, network error, locked file) → auto-retry
  - Persistent (validation error, missing dependency) → fail immediately
- [x] **AC3:** Retry policy: Exponential backoff (1s, 2s, 4s), max 3 attempts
- [x] **AC4:** Configuration in `.aiox-core/core-config.yaml` → `gates.retry_policy` (tunable)
- [x] **AC5:** All retries logged to `.aiox/gate-logs/art-gates-retries.jsonl` with timestamp, attempt #, reason, outcome

## Scope

## Risks

- **Risk:** Retry logic masks real issues (gate always succeeds after N attempts)
- **Mitigation:** Log all retries; monitor retry frequency in metrics
