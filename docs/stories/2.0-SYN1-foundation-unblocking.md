# Story 2.0-SYN1: SYNAPSE Foundation Unblocking

**Epic:** PHASE 4 — AIOX Architectural Refoundation  
**Status:** InProgress  
**Story Points:** 5 (critical fix)  
**Priority:** BLOCKING  
**Agent:** @dev  

---

## Summary

SYNAPSE Layers 2-7 never load because `session.active_agent.id` is always null. Zero enforcement gates activate. Root cause: no mechanism tracks agent activation. This story implements agent activation tracking + retroactive story enforcement gate to prevent future Art. III violations.

---

## Acceptance Criteria

- [ ] **AC1:** Agent activation tracker hook detects `@agent-name` in prompt and updates session
- [ ] **AC2:** Hook integrated in UserPromptSubmit before synapse-engine (so session is ready)
- [ ] **AC3:** Pre-commit story enforcement gate blocks commits without story linkage
- [ ] **AC4:** Gate enforced at git commit time (blocks Art. III violations)
- [ ] **AC5:** Framework files (.aiox-core/, .synapse/, .claude/hooks/) exempted from story requirement
- [ ] **AC6:** Next agent invocation triggers session update → L2+ layers load → metrics show `layersLoaded >= 3`

---

## Technical Details

### Agent Activation Tracking
- **File:** `.claude/hooks/agent-activation-tracker.cjs`
- **Trigger:** UserPromptSubmit hook chain (before synapse-engine)
- **Logic:** Extract agent ID from `@agent-name` or `/AIOX:agents:agent-name`, update session.active_agent.id
- **Result:** SYNAPSE L2 Agent Layer now has context to load

### Story Enforcement Gate
- **File:** `.claude/hooks/pre-commit-story-enforcement.cjs`
- **Trigger:** Pre-commit hook on `git commit*`
- **Logic:** Verify all staged files are linked to a story (File List or framework exempt)
- **Result:** Blocks commits that violate Art. III before they reach repository

### Integration
- **File:** `.claude/settings.json`
- **Changes:** Add hooks to UserPromptSubmit (agent tracker) and PreCommit (story gate)
