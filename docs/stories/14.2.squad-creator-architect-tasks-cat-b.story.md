## Story Statement

**As a** framework developer,  
**I want to** create 3 missing task files that are documented in official AIOX but not implemented,  
**so that** commands `*analyze-impact` (@architect), `*generate-skills` (@squad-creator), and `*generate-workflow` (@squad-creator) have executable task definitions.

---

## Problem

Research confirmed 3 commands documented in official docs have no corresponding task files:
- `@architect *analyze-impact` (`.aiox-core/development/tasks/analyze-impact.md` — missing)
- `@squad-creator *generate-skills` (`.aiox-core/development/tasks/generate-skills.md` — missing)
- `@squad-creator *generate-workflow` (`.aiox-core/development/tasks/generate-workflow.md` — missing)

These are not large new features; they follow the same task structure as existing neighbors (`architect.md` tasks, `squad-creator.md` tasks).

---

## Acceptance Criteria

- [ ] **AC1:** Create `.aiox-core/development/tasks/analyze-impact.md`  
      - Task: Assess impact of architectural/code changes (scope, dependencies, risk level)  
      - Follow structure of `architect.md`'s dependencies (template: `create-plan.md`, similar phases)  
      - Include frontmatter (name, description, inputs, outputs, execution steps)  
      - Inputs: change proposal (file paths, description); Output: impact report (scope, affected modules, risk) 
      
- [ ] **AC2:** Create `.aiox-core/development/tasks/generate-skills.md`  
      - Task: Generate Claude Code skills from squad definitions (SKILL.md files)  
      - Inputs: squad.yaml path, agent list  
      - Output: generated SKILL.md files in `.claude/skills/AIOX/agents/<agent>/SKILL.md`  
      - Follow structure similar to `squad-creator.md`'s other generation tasks

- [ ] **AC3:** Create `.aiox-core/development/tasks/generate-workflow.md`  
      - Task: Generate workflow YAML from squad specifications  
      - Inputs: workflow name, phases, agents, tasks  
      - Output: `.aiox-core/development/workflows/<name>.yaml`  
      - Follow structure similar to `generate-skills.md`

- [ ] **AC4:** Add new commands to agent definitions  
      - `.aiox-core/development/agents/architect.md` → add `*analyze-impact` to `commands:` + `analyze-impact.md` to `dependencies.tasks`  
      - `.aiox-core/development/agents/squad-creator.md` → add `*generate-skills` and `*generate-workflow` to `commands:` + to `dependencies.tasks`

- [ ] **AC5:** Test: Run `*help` for each modified agent, verify commands listed

## Dev Notes for @dev Implementation

### File Templates / Patterns to Study
- `@architect` task patterns: `.aiox-core/development/tasks/create-plan.md`, `create-doc.md`, `execute-checklist.md`
- `@squad-creator` task patterns: `.aiox-core/development/tasks/create-squad.md` (if exists), check `squad-creator.md` dependencies

### Structure Required
Each new task file must include:
```
---
name: <task-name>
description: <1-line description>
inputs: [<list of input parameters>]
outputs: [<list of outputs>]
---

## Pre-Conditions
[blocking checks before execution]

## Execution Steps
[numbered steps or phases]

## Post-Conditions
[validation after execution]

## Acceptance Criteria
[AC1-ACn pass/fail checks]
```

### Quality Gates
- No Python/JavaScript code in task files (task files are YAML specs, not code)
- All fields populated (no skipped sections)
- Grammar/spelling reviewed
