## Summary

Build CLI entry point `kairos-monitor` with subcommands. Read config from `.clauderc` (or environment variables). Implement:
- `kairos-monitor --listen 127.0.0.1:8080` — start server
- `kairos-monitor status` — show server health
- `kairos-monitor activate @agent story-id` — trigger workflow
- `kairos-monitor logs [--tail 100]` — stream audit logs

**Why:** G2 in PRD: CLI-first. Constitution Art. I requires all functionality available via CLI before UI.

---

## Acceptance Criteria

- [ ] Create `bin/kairos-monitor.js` (executable):
  - Parse command-line arguments (yargs or commander)
  - Read `.clauderc` for config: `[kairos-monitor]` section
  - Default values: `listen=127.0.0.1:8080`, `token=Bearer {random}` on first run

- [ ] Commands:
  ```bash
  kairos-monitor --listen 127.0.0.1:8080
    → start Monitor API server
    
  kairos-monitor status
    → GET /health, show: server up/down, latency, session count
    
  kairos-monitor activate @dev story-1.1 [--auto-start]
    → POST /workflow/start, show: workflow_id, status
    
  kairos-monitor logs [--tail 100]
    → stream audit log, follow mode if no --tail
  ```

- [ ] Config file `.clauderc`:
  ```ini
  [kairos-monitor]
  listen = 127.0.0.1:8080
  token_bearer = sk-...
  log_level = info
  archival_enabled = true
  ```

- [ ] Environment variable overrides:
  - `KAIROS_MONITOR_LISTEN=127.0.0.1:9090`
  - `KAIROS_MONITOR_TOKEN=sk-xxx`
  - `KAIROS_MONITOR_LOG_LEVEL=debug`

- [ ] Help text:
  ```bash
  kairos-monitor --help
  ```

- [ ] First-run setup: generate Bearer token, save to `.clauderc` (one-time)

---

## Implementation Notes

- **Constitution:** Art. I — CLI is primary interface
- **No UI required:** all features work via CLI alone
- **.clauderc:** use INI parser, create if missing on first run
- **Token:** save Bearer token securely (never log it)
