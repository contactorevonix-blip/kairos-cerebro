# Story 4.3 — Logging Format Standardization

**Epic:** EPIC-4 (CLI + Config)  
**Status:** Draft  
**Complexity:** 8sp  
**Assignee:** @dev  
**Created:** 2026-06-04

---

## Summary

Standardize all logging output (audit logs, API logs, CLI output) to structured JSON format. All logs written to `.aiox/logs/kairos-monitor-{YYYY-MM-DD}.log` (rotated daily). CLI commands also pretty-print to console.

**Why:** Observability + parsing. JSON logs enable downstream analytics, alerting, debugging.

---

## Acceptance Criteria

- [ ] Create `src/logging/Logger.ts`:
  - Structured logging with fields: timestamp, level, message, context
  - Levels: DEBUG, INFO, WARN, ERROR, CRITICAL
  - Output: both file (JSON-L) and console (pretty-printed)

- [ ] Log entry format:
  ```json
  {
    "timestamp": "ISO8601",
    "level": "INFO",
    "message": "Session started",
    "context": {
      "session_id": "uuid",
      "agent": "@dev",
      "story": "1.1"
    }
  }
  ```

- [ ] File rotation:
  - Daily at 00:00 UTC
  - Filename: `.aiox/logs/kairos-monitor-{YYYY-MM-DD}.log`
  - Gzip older files: `.aiox/logs/kairos-monitor-{YYYY-MM-DD}.log.gz`

- [ ] Console output (pretty-printed):
  ```
  2026-06-04 14:32:01 [INFO] Session started (session_id=abc123, agent=@dev)
  ```

- [ ] Log levels configurable:
  - `.clauderc`: `log_level = debug` (default: info)
  - `KAIROS_MONITOR_LOG_LEVEL=debug`

- [ ] Sensitive data filtering:
  - Tokens never logged (mask if found)
  - API responses truncated (max 500 chars)

- [ ] Tests: verify JSON format, rotation, filtering

---

## Implementation Notes

- **JSON-L format:** one entry per line, easy to parse
- **No secrets:** logger checks for common patterns (token, password, key) and masks
- **Rotation:** daily + gzip keeps disk usage low
