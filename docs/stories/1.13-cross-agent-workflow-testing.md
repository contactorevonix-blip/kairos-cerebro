# Story 1.13 — Cross-Agent Workflow Testing

**Epic:** PHASE 2 — Framework Compliance & Automation Validation  
**Story ID:** 1.13  
**Effort:** 7 story points  
**Status:** Ready for Review  

---

## Goal

End-to-end testing of all 4 primary AIOX workflows (SDC, QA Loop, Spec Pipeline, Brownfield Discovery) to validate cross-agent coordination, context propagation, and workflow gates function correctly.

## Business Value

Workflows are the backbone of AIOX development process. Testing ensures all agents can coordinate without friction and handoffs preserve context. Critical for framework reliability.

## Prerequisites

- ✅ Story 1.8 — AIOX Framework Compliance Audit (DONE)
- ✅ All 4 workflows operational (Stories 1.4-1.7)

## Scope

### In Scope
- Test SDC (Story Development Cycle): @sm → @po → @dev → @qa → @devops
- Test QA Loop: review-fix iterations with @qa and @dev
- Test Spec Pipeline: gather-assess-research-write-critique-plan phases
- Test Brownfield: 10-phase discovery workflow
- Validate handoff protocol compliance
- Validate context preservation across agents

### Out of Scope
- Individual agent unit testing (agent-level tests)
- Performance testing (for separate epic)
- User-facing workflow documentation (separate story)

## Risks

1. **Complex Interactions** — Workflows involve 5+ agents, hard to debug
   - *Mitigation:* Structured test cases, detailed logging
2. **Timing Issues** — Handoffs may fail intermittently
   - *Mitigation:* Run tests multiple times, capture logs

## Technical Notes

**Workflows to Test:**

1. **SDC (Story Development Cycle)** — 5 phases
   - Phase 1: @sm creates story
   - Phase 2: @po validates
   - Phase 3: @dev implements
   - Phase 4: @qa gates
   - Phase 5: @devops pushes

2. **QA Loop** — Iterative review-fix
   - @qa reviews → verdict → @dev fixes → re-review (max 5)

3. **Spec Pipeline** — 6 phases
   - Gather → Assess → Research → Write Spec → Critique → Plan

4. **Brownfield** — 10 phases
   - Data collection → Draft/Validation → Finalization

**Test Framework:**
```bash
npm test -- --testPathPattern="workflow"
# Manual integration testing for handoff validation
```

**Output Files:**
- `docs/WORKFLOW-TEST-RESULTS.md` — Test report
- `docs/WORKFLOW-COVERAGE.json` — Coverage matrix
- `tests/integration/workflows/` — Test files
