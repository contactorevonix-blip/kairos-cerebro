## Story
**As a** Pedro,
**I want** um script que processa o conteúdo raw (GitHub + YouTube) e gera ficheiros MDX estruturados por lição,
**so that** o Fumadocs possa renderizá-los como páginas e as lições tenham estrutura consistente desde o início.

## Epic Context
- **Epic:** EPIC-1 — Foundation: Setup + Data Pipeline
- **PRD Reference:** Section 5, Story 1.4
- **Complexidade:** M (3 pts)
- **Depends on:** Story 1.1 (Next.js + Fumadocs), Story 1.2 (GitHub content), Story 1.3 (YouTube content)

## Acceptance Criteria

1. Script `scripts/build-content-db.js` executável com `node scripts/build-content-db.js`
2. Processa ficheiros de `content/raw/github/` e `content/raw/youtube/`
3. Gera ficheiros `.mdx` em `content/lessons/` com frontmatter correcto:
   ```yaml
   ---
   title: "Título da Lição"
   description: "Descrição breve"
   module: 0  # módulo número
   lesson: 1  # lição número
   level: foundation  # foundation|build-cycles|agent-creation|advanced|master
   agents: ["@sm", "@dev"]  # agentes envolvidos
   workflow: "SDC"  # workflow principal
   duration: "30 min"  # estimativa
   sources: ["github:agents/sm.md", "youtube:G2KcYWSyIbI"]
   ---
   ```
4. Gera as 3 lições do Módulo 0 como MDX base (estrutura mínima, conteúdo a expandir em EPIC-2):
   - `content/lessons/module-0/l0.1-install-aiox.mdx`
   - `content/lessons/module-0/l0.2-configure-claude-code.mdx`
   - `content/lessons/module-0/l0.3-12-agents-overview.mdx`
5. Cada MDX gerado renderiza sem erro no Fumadocs (`npm run build` passes)
6. Script imprime log: lições criadas, fontes usadas, erros

## Integration Verification

**IV1:** `npm run build` passa após execução do script
**IV2:** `content/lessons/module-0/` contém 3 ficheiros `.mdx`
**IV3:** Cada MDX tem frontmatter válido (validar com gray-matter ou similar)
**IV4:** localhost:3000/lessons/module-0/l0.1-install-aiox carrega sem 404
