# Story 8.2.1: G6 CI/CD Gate Implementation

**Status:** Ready
**Story ID:** 8.2.1
**Created:** 2026-06-11
**Epic:** EPIC-8 — Framework Evolution (Phase 2: IDS Enhancement)
**PRD Source:** `docs/prd/epic-8/phase-2-ids.md#821-g6-cicd-gate-implementation-2sp`
**Story Points:** 2

## Story

**As a** framework maintainer,
**I want** an automated G6 CI/CD gate that validates entity registry integrity on every push,
**so that** new artifacts cannot enter the codebase without being registered, keeping the IDS registry trustworthy and enforcement automatic instead of manual.

---

## Acceptance Criteria

1. Git hook: `.husky/pre-push` runs G6 gate (block push if registry invalid)
2. CI task: `.aiox-core/development/tasks/ids-g6-gate.md` (formal definition)
3. Validation checks:
   - All new files registered in entity-registry.yaml
   - No circular dependencies in entity graph
   - All referenced patterns exist (no broken links)
4. Override mechanism: `--skip-ids-g6 --reason "..."` (logged for audit)
5. Performance: Gate runs <10s on full registry

## Risks

- **Risk:** G6 false positives (legitimate new files blocked) could halt all pushes for the team.
- **Mitigation:** Ship with `--skip-ids-g6 --reason "..."` override from day one (AC4), and validate against a sample of recent commits before enabling as a hard block.

- **Risk:** Performance target (<10s) may not hold once the registry grows past its current 823 entities (PRD/research figures cite 474; current `entity-registry.yaml` already shows `entityCount: 823`).
- **Mitigation:** Benchmark against the live registry (823 entities), not the PRD's stale baseline, and design for incremental/streaming validation if needed.
