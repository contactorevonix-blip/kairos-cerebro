# Story 8.2.8: Documentation + Training

**Status:** Ready

## Story

**As a** developer or agent new to the IDS system (G1-G6, REUSE>ADAPT>CREATE, registry, scoring, dashboard),
**I want** a complete IDS guide, API reference, and a short training deck,
**so that** I can understand and correctly use the entire Phase 2 IDS system (8.2.1-8.2.7) without having to read 7 separate story files and the underlying source code.

---

## Acceptance Criteria

1. File: `docs/guides/ids-guide.md`
   - Article IV-A explained (REUSE>ADAPT>CREATE)
   - When to create vs adapt (decision tree)
   - How to register entities (CLI command)
   - G1-G6 gates explained (what triggers them)
2. File: `docs/architecture/ids-api.md`
   - REST API endpoints (GET /ids/entity, GET /ids/score, etc.)
   - Entity registry format (YAML schema)
   - Change log format (JSONL schema)
3. Slides: 10-minute IDS overview (Markdown + speaker notes)

## Risks

- **Risk:** AC2 mentions "REST API endpoints (GET /ids/entity, GET /ids/score, etc.)" — but Story 8.2.4 flagged that no REST API framework was confirmed for the AIOX core (Article I is CLI First). If 8.2.4/8.2.7 land as CLI commands instead of REST endpoints, AC2 as literally written would document an API that doesn't exist.
- **Mitigation:** Document whatever was ACTUALLY implemented in 8.2.1-8.2.7 (CLI commands, file formats, or REST endpoints — whichever @architect decided). AC2's heading "ids-api.md" can document a CLI-based "API" (command reference) if that's what was built — Article IV (No Invention) means this story documents reality, not the PRD's literal `GET /ids/...` wording if that wasn't built.

- **Risk:** This story can't be drafted with concrete content yet because 8.2.1-8.2.7 are all still in `Draft` status with multiple open architecture questions (flagged throughout this batch).
- **Mitigation:** This story's Tasks/Subtasks below define the documentation STRUCTURE based on the PRD's AC, but @dev implementing this story MUST verify each documented behavior against the actual 8.2.1-8.2.7 implementations at the time 8.2.8 is picked up — not against this story's drafted assumptions.
