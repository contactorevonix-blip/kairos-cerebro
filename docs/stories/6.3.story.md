## Summary

Consolidate parallel configuration in `.claude/settings.json` and `.claude/settings.local.json`. The two files currently maintain overlapping configurations with semantic differences (different hooks, different timeouts, different async flags, absolute vs relative paths). Consolidate into single source of truth with clear precedence rules and document in CLAUDE.md.

**Impact:** CRITICAL — Configuration divergence risks silent bugs and maintenance confusion.

---

## Problem Statement

**Current State:**
- `.claude/settings.json` and `.claude/settings.local.json` both exist with overlapping content
- Same hook events registered with different matchers/orderings
- `.claude/settings.local.json` uses absolute paths (e.g., `C:\\Users\\lealp\\KAIROS_CEREBRO\\.claude\\hooks\\...`) instead of relative
- Different timeout values (pre-commit-lint: 15s in settings.json, 10s in settings.local.json)
- Different `async` flags between files
- No documented precedence rule: which file wins?

**What's Missing:**
- Single source of truth
- Clear precedence rules documented
- Consistent path handling (relative vs absolute)
- Consistent timeout/async configuration

**Source:** AIOX-SYNC-AUDIT-2026-06-10.md, Gap 1.3 (lines 67–89)
