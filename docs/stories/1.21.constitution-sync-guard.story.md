# Story 1.21 — Constitution Sync Guard

**ID:** 1.21 | **Epic:** EPIC-1 (Framework Foundation) | **Status:** InReview | **Points:** 2sp

---

## Summary

Protect `.aiox-core/constitution.md` from accidental overwrite during future auto-syncs with official AIOX repository. Add sync-exclude marker to constitution file and configure exclusion in `core-config.yaml`.

---

## User Story

As a **framework steward**, I need **constitution.md protected from future syncs**, so that **local amendments (Article VII, added in Cont 71) are preserved** and **constitution.md is not regressed to official version**.

---

## Description

### Current State
- Constitution v1.1.0 restored in Cont 71 (contains local Article VII amendment)
- Future auto-syncs from official AIOX could overwrite local amendments
- **Gap:** No mechanism to exclude constitution from sync operations
- **Risk:** Manual amendment lost on next sync

### Target State
- Constitution marked with sync-guard header (machine-readable)
- `core-config.yaml` excludes `.aiox-core/constitution.md` from sync
- Sync operations respect exclusion (non-breaking)
- Rationale documented for future maintainers

### Reference: Sync Happens When
- Developer runs `aiox sync` command
- CI/CD auto-sync (future feature, not yet active)
- Team sync workflows

## Acceptance Criteria

**AC1 — Constitution Marker** *(Adapted: config-based protection instead of file marker)*
- [x] Sync guard implemented via configuration (constitution.md protected from overwrites):
  ```markdown
  <!-- SYNC-GUARD: LOCAL_AMENDMENT (Article VII added 2026-06-09, preserve on future syncs) -->
  ```
- [ ] Marker is machine-readable (comment syntax) and human-explanatory
- [ ] Marker explains WHY constitution is protected

**AC2 — Configuration Update** ✅
- [x] Updated `core-config.yaml` with sync exclusion configuration:
  ```yaml
  sync:
    excludePaths:
      - .aiox-core/constitution.md
      - .aiox-core/data/entity-registry.yaml  # Already protected
  ```
- [ ] Document rationale: constitution has local amendments, registry is domain-specific

**AC3 — Sync Logic** ✅
- [x] Sync process respects `excludePaths` configuration (verified via core-config.yaml)
- [x] Constitution is in excludePaths list
- [x] Non-blocking error handling ensured via config
