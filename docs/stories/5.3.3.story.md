## Summary

Design and implement persistent registry for storing context state, gap analysis results, and validation status. Registry enables Phase 5 (IDS-CHECK) queries and Phase 10 (PERSISTENCE) updates. JSONL format with in-memory indices for fast lookups.

**Impact:** CRITICAL — Enables pattern reuse detection (REUSE > ADAPT > CREATE).

## Technical Context

**Registry Schema (JSONL):**
```json
{
  "id": "ctx-2026-06-09-001",
  "timestamp": "2026-06-09T14:30:00Z",
  "session_id": "session-abc123",
  "phase": 10,
  "intent": "user input statement",
  "gaps_detected": ["context-X", "context-Y"],
  "context_sources": {
    "registry": ["entity-123", "entity-456"],
    "project_md": ["PROJECT.md#section1"],
    "inference": ["deduced-context"]
  },
  "validation_score": 0.95,
  "validated": true
}
```

**Indices:**
- `intent_index`: intent → [entry_ids]
- `date_index`: date → [entry_ids]
- `pattern_index`: gap_pattern → [entry_ids]

**File Structure:**
```
.aiox-core/data/
├── context-registry.jsonl
├── context-registry-schema.md
└── context-registry-indices.json (in-memory, recreated on load)
```
