# Story 13.3: ideSync Drift Verification

**Epic:** EPIC-13  
**Story ID:** 13.3  
**Status:** Ready  
**Effort:** 3 sp  
**Owner:** @devops (CI/CD gate)  
**Priority:** 🟠 HIGH (Phase 2)

---

## Story Statement

**As a** DevOps engineer,  
**I want** an automated pre-commit gate that detects when framework components drift from their source,  
**so that** divergence between canonical sources (L2) and targets (L3 ideSync locations) is caught immediately.

---

## Problem

`core-config.yaml` declares ideSync targets (`.codex/`, `.antigravity/`, `.claude/skills/`, etc.), but **no pre-commit validation** ensures they stay in sync with source.

**Risk:** Agents, rules, or workflows can silently diverge. Changes in L2 aren't propagated to L3 targets.

**Example:** Delete agent from L2 → source is gone, but `.claude/skills/AIOX/agents/{name}/` still exists → stale code runs.

---

## Acceptance Criteria

- [ ] **AC1:** Create `npm run verify-ideSync` script that compares L2 sources against all declared ideSync targets
- [ ] **AC2:** Script detects 3 types of drift:
  - Missing targets (source exists, target doesn't)
  - Extra targets (target exists, source doesn't)
  - Stale files (target timestamp older than source)
- [ ] **AC3:** Add script to pre-commit hook or `npm test` so it runs on every commit
- [ ] **AC4:** Script reports drift in CI/CD gate; merge fails if drift detected
- [ ] **AC5:** Error message includes: expected vs. actual files, sources, targets, remediation steps

---

## Technical Context

### ideSync Configuration

In `core-config.yaml`, ideSync targets might be listed as:
```yaml
ideSync:
  enabled: true
  targets:
    - path: .codex/agents
      source: .aiox-core/development/agents
    - path: .antigravity/agents
      source: .aiox-core/development/agents
    - path: .claude/skills/AIOX/agents
      source: .aiox-core/development/agents
```

### Verification Logic

**Pseudocode:**
```
FOR each ideSync target:
  source_files = list(.aiox-core/development/agents)
  target_files = list(target.path)
  
  FOR each source_file:
    IF NOT EXISTS target.path/source_file:
      REPORT: "Missing target: {target.path}/{source_file}"
  
  FOR each target_file:
    IF NOT EXISTS .aiox-core/development/agents/target_file:
      REPORT: "Extra target (source deleted): {target.path}/{target_file}"
  
  IF source_file.mtime > target_file.mtime:
    REPORT: "Stale target: {target.path}/{target_file} (source updated 2h ago)"
```

## Tasks / Subtasks

1. **Parse ideSync Configuration**
   - Read `core-config.yaml`
   - Extract `ideSync.targets` (or similar key)
   - Document source ↔ target mappings
   - Link: AC1

2. **Implement Drift Detection Logic**
   - Create script: `scripts/verify-ideSync.js`
   - Read source directory
   - Read target directory
   - Compare files (existence, timestamps)
   - Generate report
   - Link: AC1, AC2

3. **Add 3-Type Drift Detection**
   - Type 1: Missing targets (source exists, target doesn't)
   - Type 2: Extra targets (target exists, source doesn't)
   - Type 3: Stale targets (source modified later than target)
   - Link: AC2

4. **Format Error Output**
   - Clear message: "Drift detected in ideSync targets:"
   - List each drift with type
   - Include file paths
   - Suggest remediation (e.g., "re-run generator", "delete stale files")
   - Link: AC5

5. **Integrate into Pre-Commit Hook**
   - Update `.husky/pre-commit` to call script
   - Or create `.claude/hooks/verify-ideSync.cjs`
   - Hook should fail commit if drift found
   - Link: AC3

6. **Integrate into CI/CD Gate**
   - Add to `.github/workflows/` test/merge job
   - Script runs before merge
   - Blocks merge if drift detected
   - Reports drift in PR checks
   - Link: AC4

7. **Add npm Script**
   - `package.json` → add `scripts.verify-ideSync: "node scripts/verify-ideSync.js"`
   - Callable as `npm run verify-ideSync`
   - Link: AC1
