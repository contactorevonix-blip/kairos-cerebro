# Story 13.7: Implement Memory Persistence

**Story ID:** 13.7  
**Epic:** EPIC-13 (Full Context Determinism)  
**Status:** Draft  
**Points:** 6sp  
**Owner:** @dev  
**Dependencies:** 13.1, 13.2, 13.3, 13.4, 13.5, 13.6

## Description

Implement persistent memory system that captures full context state to `.synapse/sessions/{sessionId}.json`. After all 8 layers load successfully, snapshot the complete state atomically so that agent activation can recover 100% context from a prior session with <100ms latency.

## Acceptance Criteria

- [ ] On successful load of all layers (13.1-13.6), snapshot state to `.synapse/sessions/{sessionId}.json`
- [ ] Session ID format: `{date}-{agentId}-{index}` (e.g., `2026-06-17-dev-001`)
- [ ] Atomic snapshots (transaction semantics: all-or-nothing)
- [ ] Snapshot includes: Constitution, Global rules, all 8 layers, cache metadata
- [ ] On agent activation, load persisted context if available and fresh (TTL not expired)
- [ ] Recovery from persisted context succeeds >99% of time
- [ ] TTL validation: purge sessions >30 days old
- [ ] Logs snapshot + recovery events to `.aiox/context-load-logs/`
- [ ] All 4 pre-commit gates PASS
