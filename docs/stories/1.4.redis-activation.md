**I want** activar o adapter Redis para o reputation graph no Railway,
**so that** a inteligência cross-tenant funcione correctamente em produção e o sistema esteja pronto para escala horizontal.

## Dev Notes

**Zero mudança de código.** O adapter Redis já está implementado em `packages/reputation-graph/adapters/`.

**Variáveis a definir (só 2):**
```
KAIROS_RG_ADAPTER=redis
KAIROS_REDIS_URL=redis://default:<password>@<host>:<port>
```

**Como o adapter funciona:**
- `packages/reputation-graph/adapters/` — tem implementações JSON (default) e Redis
- A variável `KAIROS_RG_ADAPTER` controla qual adaptador é carregado no startup
- Se `KAIROS_RG_ADAPTER` não estiver definido → usa JSON on-disk (comportamento actual)

**Contexto de negócio:** Activar Redis resolve dois débitos de uma vez:
- DB-007: Redis não activo em produção
- DB-001: JSON sem locking multi-processo (Redis é thread-safe por natureza)

**Verificação de que o adapter está a funcionar:**
```bash
# Ver nos logs do Railway após deploy
grep -i "redis" # deve aparecer "reputation graph: using redis adapter"
```

**Custo Railway Redis:** ~$5/mês (plano Hobby). Adequado para o volume actual (0 tenants).
