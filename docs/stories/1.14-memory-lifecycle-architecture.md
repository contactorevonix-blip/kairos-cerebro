## Dev Notes

### Memory Lifecycle Phases
1. **Discovery** — Detect new agent (from AIOX agents registry)
2. **Creation** — Auto-create MEMORY.md in `.claude/agents/{id}/`
3. **Population** — Populate from agent personality, skills, memory imports
4. **Synchronization** — Keep in sync with agent updates
5. **Validation** — Ensure structure + required fields

### Key Concepts
- **Memory Requirements** — Fields that MUST be populated (from agent definition)
- **Rastreabilidade** — Log WHO updated WHAT and WHY
- **Conflict Lock** — Prevent concurrent updates during sync
- **Discovery Source** — Agent registries (agents.yaml, registry, skill definitions)

### Constraints
- MEMORY.md must follow frontmatter structure: `name`, `description`, `metadata.type`
- No memory content longer than recommended (token budget)
- All updates must have audit trail
