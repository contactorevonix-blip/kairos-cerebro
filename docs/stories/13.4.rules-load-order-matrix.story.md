## Story Statement

**As a** framework architect,  
**I want** a single document that explains when and how each AIOX rule loads,  
**so that** developers understand the precedence and don't get confused about conflicting rules.

---

## Problem

Rules exist in multiple places with unclear precedence:
- `.claude/rules/*.md` — loaded by hooks at startup
- `.aiox-core/development/workflows/*.md` — templates, loaded on-demand
- `.aiox-core/constitution.md` — Articles I–VII, always in effect

**Risk:** Developers don't know which rule applies. Is `.claude/rules/story-lifecycle.md` overridden by something in workflows? Does Constitution Article override local rules?

**Impact:** Inconsistent enforcement, developers don't follow rules they're unaware of.

## Technical Context

### Rule Loading Mechanism

**Constitution (Always):**
- Located: `.aiox-core/constitution.md`
- Articles: I–VII (CLI First, Agent Authority, Story-Driven, No Invention, Quality First, Absolute Imports, others)
- Status: NON-NEGOTIABLE (enforced by gates in `.claude/hooks/`)

**Rules (At Startup):**
- Located: `.claude/rules/*.md` (e.g., `story-lifecycle.md`, `agent-authority.md`, etc.)
- Trigger: Claude Code startup, or when matching `paths:` patterns in frontmatter
- Load Order: Probably alphabetical or by dependency, check hook code
- Override: Can conditionally override Constitution? (investigate)

**Workflows (On-Demand):**
- Located: `.aiox-core/development/workflows/*.md` or `.aiox-core/development/tasks/*.md`
- Trigger: Agent invokes workflow (e.g., `@dev *develop`)
- Load Order: Task runner loads on-demand
- Override: Unlikely to override Constitution, but may refine rules

### Expected Hierarchy
```
Constitution (Articles I–VII) — NEVER overridden
  ↓ (enforced by gates)
.claude/rules/*.md — can refine, not contradict
  ↓ (loaded at startup)
.aiox-core/development/workflows/*.md — task-specific guidance
  ↓ (loaded on-demand)
Agent decisions (based on context)
```
