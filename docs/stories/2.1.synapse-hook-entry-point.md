## Story

**As a** developer a usar o KAIROS_CEREBRO,
**I want** que o SYNAPSE engine injete regras contextuais em cada prompt automaticamente,
**so that** o Claude receba sempre o contexto do projecto (arquitectura, segurança, convenções) sem eu ter de repetir tudo manualmente.

---

## Contexto Técnico

O SYNAPSE (Synkra Adaptive Processing & State Engine) está **100% implementado** em `.aiox-core/core/synapse/` mas o ficheiro de entrada do hook está em falta.

**Arquitectura existente:**
```
.aiox-core/core/synapse/engine.js          ← SynapseEngine (✅ existe)
.aiox-core/core/synapse/runtime/hook-runtime.js  ← resolveHookRuntime + buildHookOutput (✅ existe)
.aiox-core/core/synapse/layers/l0-l7.js    ← 8 layers (✅ existe)
.synapse/manifest                          ← domínios activos (✅ existe)
.synapse/global, kairos, devops, quality   ← regras (✅ existem)
.claude/hooks/synapse-engine.cjs           ← FALTA — este é o único ficheiro em falta
```

**Domínios activos e o que injectam:**
- `global` (always-on): segurança, ferramentas correctas, convenções de código
- `kairos` (always-on): Railway URL, env vars críticas, scripts do projecto
- `quality` (always-on): zero abstrações prematuras, sem error handling impossível
- `devops` (trigger: agente @devops activo): regras de git/deploy
- `ids` (trigger: *create/*modify): REUSE > ADAPT > CREATE

**Referência em settings.json (linha 57):**
```json
{
  "type": "command",
  "command": "node \".claude/hooks/synapse-engine.cjs\"",
  "timeout": 10,
  "statusMessage": "Synapse Engine a processar..."
}
```
O hook está registado — só precisa do ficheiro.

## Acceptance Criteria

1. Ficheiro `.claude/hooks/synapse-engine.cjs` criado e funcional
2. O hook lê o evento `UserPromptSubmit` do Claude Code (JSON via stdin)
3. Chama `resolveHookRuntime(input)` de `.aiox-core/core/synapse/runtime/hook-runtime.js`
4. Chama `engine.process(prompt, session, config)` com `_hookBootTime`
5. Escreve `buildHookOutput(result.xml)` para stdout
6. **Nunca bloqueia** — exit 0 sempre, mesmo em caso de erro
7. Teste manual: `echo '{"message":"test","cwd":"C:/Users/lealp/KAIROS_CEREBRO","session_id":"test-123"}' | node .claude/hooks/synapse-engine.cjs` devolve JSON válido com `hookSpecificOutput`
8. Timeout de 10s no settings.json é suficiente (pipeline corre em < 100ms — `PIPELINE_TIMEOUT_MS = 100`)
