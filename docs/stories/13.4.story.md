# Story 13.4: Load Task Specifications

**Story ID:** 13.4  
**Epic:** EPIC-13 (Full Context Determinism)  
**Status:** Ready

## Description

Load all ~50 task specifications from `.aiox-core/development/tasks/*` with inputs, outputs, and quality gates validation. Tasks are the atomic unit of work; each task must declare its interface and success criteria deterministically.

## Acceptance Criteria

- [ ] Loads all ~50 task specs from `.aiox-core/development/tasks/*` per L3 schema (see Technical Notes)
- [ ] Parses task structure (inputs, outputs, pre/post-conditions, quality gates)
- [ ] Validates task interface (inputs/outputs type-checked against schema)
- [ ] Builds dependency graph (task→prerequisites mapping) + detects circular deps
- [ ] Topological sort (Kahn algorithm) validates optimal task execution order
- [ ] Caches task specs + dependency graph in `.synapse/task-cache/{task-id}.json`
- [ ] Cache TTL: 1 hour (configurable via `.aiox-core/data/cache-config.yaml`)
- [ ] On task execution, loads cache if valid, else reloads from disk
- [ ] Detects and reports missing required task gates (BLOCK if any missing)
- [ ] Logs cache hit/miss + validation results to `.aiox/context-load-logs/{date}.jsonl`
- [ ] All 4 pre-commit gates PASS (lint ✅, typecheck ✅, test ✅, CodeRabbit pending)

## Technical Notes

**L3 Schema for Task Specification (`.synapse/task-cache/{task-id}.json`):**

```json
{
  "id": "task-create-story",
  "name": "Create Next Story",
  "version": "1.0.0",
  "description": "Create new story from epic/PRD context",
  "agent": "@sm",
  "inputs": {
    "epic_id": { "type": "string", "required": true, "description": "EPIC-N reference" },
    "prd_context": { "type": "object", "required": true, "description": "Sharded PRD section" }
  },
  "outputs": {
    "story_file_path": { "type": "string", "description": "docs/stories/N.M.story.md" },
    "story_id": { "type": "string", "description": "N.M format" }
  },
  "pre_conditions": [
    "epic_id must be valid EPIC-N format",
    "prd_context must contain scope + AC template"
  ],
  "post_conditions": [
    "Story file created at docs/stories/{epic}/{story}.story.md",
    "Status field set to 'Draft'",
    "File List populated",
    "All ACs structured with checkboxes"
  ],
  "quality_gates": [
    "story-ac-validation",
    "framework-boundary-check",
    "no-invention-audit"
  ],
  "dependencies": [
    "story-lifecycle-rules loaded",
    "epic context available"
  ],
  "execution_time_ms": 500,
  "metadata": {
    "created_at": "2026-06-17T00:00:00Z",
    "determinism_score": 100,
    "is_critical": true
  }
}
```

**Dependency Graph + Kahn Algorithm:**
- Build directed graph: task → [required_tasks]
- Validate: No task depends on itself (cycle detection)
- Topological sort: Compute optimal execution order (Kahn algorithm)
- Output: Array of task IDs in execution order [task_1, task_2, ...]
- Use case: Parallel task execution planning

**Implementation Strategy:**
- Story 13.3: Workflow loader + DAG validation (high-level)
- Story 13.4: Task loader + Kahn sort (atomic task level)
- Story 13.5: Use Kahn output for escalation decision tree
