### Completion Notes

**AC1 (TEST-001 — HIGH):** Root cause confirmed empirically (baseline: 3 pass / 9 cancelled / exit 1). The `runContextPhasesGuarded` guard timer was `unref()`'d, so while the `HANGING_ENGINE` test awaited the 2000ms budget the event loop had no live handle → `node --test` resolved the loop early and cancelled that test + 8 cascading. **Fix (option a):** removed the guard-timer `unref()` in `agent-activation-tracker.cjs`. Safe for production because (1) the timer is always cleared in the `finally` block when the race resolves, so it can never keep the process alive beyond the function's own resolution, and (2) the standalone hook process is independently bounded by `run()`'s own `unref()`'d 3s safety timer. Verified the production hook still terminates cleanly (spawned with a real prompt → `{}` output, exit 0, 51ms). Result: hook test 12/12, full hook glob 75/75, `npm test` exit 0.

**AC2 (REQ-001 — MEDIUM):** Searched for any formal FR mandating `aiox run-context-engine` — none exists (only references are the audit artifacts themselves). AC1 of 5.3.1 reworded + checked; decision recorded in 5.3.1 Change Log (v1.3.0) citing this story.

Reconciled to a single JSON store: removed the stale `.synapse/context-registry.yaml`, updated `registryPath` and `REGISTRY-SCHEMA.md`.
