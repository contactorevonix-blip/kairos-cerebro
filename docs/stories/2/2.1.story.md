# Story 2.1 — Constitutional Gates Enforcement

**Epic:** EPIC-2 (Control Core)  
**Status:** Draft  
**Complexity:** 13sp  
**Assignee:** @dev  
**Created:** 2026-06-04

---

## Summary

Implement API middleware that enforces AIOX Constitution (Articles I-VI) at the Control API boundary. Block git push, gh pr, MCP add/remove operations with 403 Forbidden. Log all gate enforcement attempts to AuditLog (Art. II requires auditability).

**Why:** Constitution is non-negotiable. Gates must be enforced at API boundary, not left to agents. FR4 in PRD requires this.

---

## Acceptance Criteria

- [ ] Create `src/control/ConstitutionalGate.ts`:
  - Middleware function for Fastify
  - Checks request against 6 gate rules (Art. I-VI)
  - Returns `403 Forbidden` with reason on violation

- [ ] Gate rules (Art. I-II exclusive):
  ```
  1. git push → BLOCK (only @devops via cli, not API)
  2. git push --force → BLOCK
  3. gh pr create → BLOCK
  4. gh pr merge → BLOCK
  5. mcp add / mcp remove / mcp configure → BLOCK
  6. ci/cd pipeline modify → BLOCK
  ```

- [ ] Response format:
  ```json
  {
    "error": "ConstitutionalViolation",
    "gate": "git_push",
    "reason": "git push is exclusive to @devops (Art. II)",
    "timestamp": "ISO8601",
    "audit_id": "uuid"
  }
  ```

- [ ] Every violation logged to AuditLog (Story 1.3) with `event_type: 'gate_enforcement'`
- [ ] Whitelist mechanism: `CONSTITUTIONAL_WHITELIST` env var (for testing, default empty)
- [ ] Tests: verify each gate blocks correctly, audit entries created

## Implementation Notes

- **Constitution:** Art. II — only @devops can push/PR, enforced here
- **No invention:** gates come from Constitution only, no extra rules
- **Logging:** every rejection is audited (compliance requirement)
