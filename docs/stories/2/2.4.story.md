## Summary

Implement rate limiting middleware for Control API to prevent abuse. Limit `POST /workflow/start` to 10 requests/minute per agent. Limit `GET` endpoints to 100 requests/minute per client. Excess requests return `429 Too Many Requests`.

**Why:** Prevent accidental or malicious API abuse. Ensure system stability under heavy load.

---

## Acceptance Criteria

- [ ] Create `src/control/RateLimiter.ts`:
  - In-memory rate limiter (no external service)
  - Token bucket algorithm (leaky bucket)
  - Per-client tracking via Bearer token or IP

- [ ] Rate limits:
  - `POST /workflow/start`: 10 req/min per agent
  - `GET /sessions`: 100 req/min per client
  - `GET /agents`: 100 req/min per client
  - `GET /tasks`: 100 req/min per client
  - `GET /hooks`: 100 req/min per client

- [ ] Response on limit exceeded:
  ```json
  {
    "error": "RateLimitExceeded",
    "retry_after_seconds": 12,
    "limit": 10,
    "window_seconds": 60
  }
  ```

- [ ] Headers:
  - `X-RateLimit-Limit: 10`
  - `X-RateLimit-Remaining: 3`
  - `X-RateLimit-Reset: 1717507200`

- [ ] Cleanup: old rate limit entries (older than window) garbage-collected
- [ ] Tests: verify limits enforced, headers correct, cleanup works

---

## Implementation Notes

- **No external service:** in-memory only (localhost only, no persistence needed)
- **Idempotency:** rate limits are per-token, not per-IP (Bearer token primary key)
- **Fallback:** if token missing, use IP address
