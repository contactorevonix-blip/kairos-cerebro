# Story 7.2: `package.json` — Adicionar Dependência `ajv-formats`

**Status:** Draft

## Summary

Adicionar a dependência NPM `ajv-formats@^3.0.1` a `package.json`. Dois módulos do codebase (`core/registry/validate-registry.js` e `core/config/config-resolver.js`) fazem `require('ajv-formats')` mas a dependência não está declarada, causando falhas silenciosas de validação.

**Impact:** MEDIUM — Validação de schema está desligada silenciosamente, caminho para dados inválidos no sistema.

---

## Problem Statement

**Current State:**
- `package.json` não contém `ajv-formats` nas dependências
- `core/registry/validate-registry.js` faz `require('ajv-formats')` (linha X)
- `core/config/config-resolver.js` faz `require('ajv-formats')` (linha Y)
- Ambos têm try/catch, falham silenciosamente se módulo em falta
- **Resultado:** Validação de schema desligada, sem reportagem de erro

**Root Cause:** Dependência foi usada no upstream mas não foi incluída no `package.json` local durante port inicial.

**Source:** AIOX-CORE-REALIGNMENT-AUDIT-2026-06-10.md, Gap A.2

---

## Acceptance Criteria

- [ ] **AC1:** Adicionar `"ajv-formats": "^3.0.1"` a `package.json` em dependencies
- [ ] **AC2:** Executar `npm install` para confirmar instalação
- [ ] **AC3:** Testar que `require('ajv-formats')` resolve sem erro: `node -e "require('ajv-formats'); console.log('OK')"`
- [ ] **AC4:** Verificar que `core/registry/validate-registry.js` e `core/config/config-resolver.js` agora carregam o módulo correctamente
- [ ] **AC5:** Considerar remover/auditar o try/catch se agora seguro (depende de contexto de uso)
