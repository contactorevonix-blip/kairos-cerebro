# Story 8.1.6: Alerts & Thresholds

**Status:** Ready
**Story ID:** 8.1.6
**Created:** 2026-06-11
**Epic:** EPIC-8 — Framework Evolution (Phase 1: Observability & Metrics)
**PRD Source:** `docs/prd/epic-8/phase-1-observability.md#816-alerts--thresholds-15sp`
**Story Points:** 1.5

---

## Story

**As a** framework maintainer,
**I want** configurable health thresholds and an on-demand alert check command,
**so that** I am warned when gate compliance, story velocity, or framework health degrade below acceptable levels.

---

## Acceptance Criteria

1. Configuration file: `.aiox-core/data/metrics-alerts.yaml`
   - Gate compliance threshold: >95% (ALLOW blocks)
   - Story velocity threshold: >1 story/day (WARN if below)
   - Framework health threshold: >90% (CRITICAL if below)
2. Alert channels: Console (CLI) + email (optional)
3. Trigger: On-demand `aiox metrics alert-check`
4. Documentation: Alert configuration guide

---
