## Story Statement

**As a** QA engineer,  
**I want** to verify that all Constitutional enforcement hooks (Art. I–VII) exist, load properly, and work as designed,  
**so that** the framework's non-negotiable principles are enforced consistently.

---

## Problem

`.aiox-core/constitution.md` declares Art. I–VII enforcement, but:
- Some hooks are fully implemented ✅
- Some degrade gracefully ⚠️
- Some may not exist or aren't loaded ❌
- Dead code may reference non-existent hooks

**Risk:** Constitution is declared but not enforced. Violations slip through.

**Impact:** Framework governance breaks; agents violate principles without detection.

---

- [ ] **AC1:** Verify each Article I–VII has a corresponding enforcement hook in `.claude/hooks/`

## Technical Context

### Constitution Articles & Enforcement

**Article I: CLI First (NON-NEGOTIABLE)**
- Hook: `.claude/hooks/enforce-cli-first.cjs` (or similar)
- Enforcement: CLI is source of truth; UI/dashboards observe only
- Gate: Likely checked at deployment time

**Article II: Agent Authority (NON-NEGOTIABLE)**
- Hook: `.claude/hooks/enforce-agent-authority.cjs`
- Enforcement: Only @devops can push; only @sm can create stories, etc.
- Gate: PreToolUse gate on `git push`, `gh pr create`

**Article III: Story-Driven Development (MUST)**
- Hook: `.claude/hooks/enforce-story-driven.cjs`
- Enforcement: No code commit without active story
- Gate: PreToolUse gate on `git commit` (degrade to WARN if story lookup fails)

**Article IV: No Invention (MUST)**
- Hook: `.claude/hooks/enforce-no-invention.cjs`
- Enforcement: Specs must trace to requirements
- Gate: PreToolUse gate on Write/Edit to `*spec.md` (degrade gracefully)

**Article V: Quality First (MUST)**
- Hook: `.claude/hooks/enforce-quality-gates.cjs`
- Enforcement: Code quality, test coverage, CodeRabbit pass
- Gate: PreToolUse gate on `git merge` (degrade to WARN if data unavailable)

**Article VI: Absolute Imports (SHOULD)**
- Hook: Linter rule, not gate
- Enforcement: Relative imports flagged as warnings in linting
- Status: SHOULD, not enforced in gates

**Article VII: Framework Boundary (MUST)**
- Hook: `.claude/hooks/enforce-framework-boundary.cjs`
- Enforcement: L1/L2 protected (deny rules in `.claude/settings.json`)
- Gate: Blocks Write/Edit to L1/L2 paths

### Hook Implementation Status

Need to verify for each:
1. File exists: `.claude/hooks/enforce-{article}.cjs`?
2. Loads correctly: No syntax errors?
3. Matches correctly: Does gate trigger on intended conditions?
4. Enforces correctly: Does it block/allow as specified?
5. Degrades gracefully: If data unavailable, does it WARN instead of BLOCK?

## Tasks / Subtasks

1. **Audit Constitution Articles I–VII**
   - Read `.aiox-core/constitution.md`
   - For each article, document:
     - Article number & title
     - Enforcement requirement (NON-NEGOTIABLE / MUST / SHOULD)
     - Expected enforcement mechanism
   - Link: AC1, AC5

2. **Verify Hook Files Exist**
   - List all `.claude/hooks/enforce-*.cjs` files
   - Map each to Constitution article
   - Report: which articles have hooks, which don't?
   - Link: AC1

3. **Test Hook Load & Syntax**
   - For each hook file:
     - Check syntax (run node with `--check`)
     - Check for missing dependencies
     - Verify it imports correctly
   - Document findings
   - Link: AC2

4. **Test Hook Trigger Conditions**
   - For each hook, test when it triggers:
     - Article II (git push): triggers on `git push`? On `gh pr create`?
     - Article III (story-driven): triggers on `git commit` without story?
     - Etc.
   - Verify matches are correct (no false negatives)
   - Link: AC2

5. **Test Hook Enforcement**
   - For each hook, test enforcement behavior:
     - BLOCK expected: does it block?
     - WARN expected: does it warn?
     - Allow expected: does it allow?
   - Test edge cases (e.g., data unavailable, malformed input)
   - Link: AC2

6. **Document Graceful Degradation**
   - For Art. III–V, check degradation logic:
     - If story lookup fails: WARN or BLOCK?
     - If quality data unavailable: WARN or BLOCK?
   - Document intended vs. actual behavior
   - Link: AC3

7. **Identify Dead Code**
   - Scan enforcement hooks for:
     - References to non-existent rules
     - References to deprecated agents
     - Unreachable code paths
   - List findings
   - Link: AC4

8. **Create Unit Tests**
   - Test file: `tests/hooks/enforcement.test.js` (or per-article files)
   - For each article:
     - Test block case (violation detected)
     - Test allow case (no violation)
     - Test graceful degradation
     - Test edge cases
   - Link: AC6

9. **Update Constitution**
   - Based on findings, update `.aiox-core/constitution.md`:
     - Add actual enforcement status (not just declared)
     - Document graceful degradation behavior
     - Link to test file
   - Link: AC5
