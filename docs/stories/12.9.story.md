### Exclusive Operations Workflow

@devops follows push/PR workflow:
1. **Pre-flight:** Verify story status = Done, all gates PASS
2. **Push Branch:** `git push origin feature/X.Y-story-name`
3. **Create PR:** `gh pr create --title '...' --body '...'` (links to story)
4. **CI/CD Check:** Await GitHub Actions completion (tests, linter, security)
5. **Review:** @devops reviews CI/CD results
6. **Merge PR:** `gh pr merge --squash` (if all checks pass)
7. **Post-merge:** Update story status, log deployment
