# Story 5.3.2: Registration Hook Integration

**Status:** Done
**Story ID:** 5.3.2
**Created:** 2026-06-09
**Epic:** Epic-5-3 — Permanent Auto-Contextualization System
**Executor:** @dev
**Quality Gate:** @architect
**Story Points:** 5

---

## Summary

Wire the auto-contextualization engine (`ContextEngine`, Story 5.3.1) into the existing agent activation hook so that Phases 1-4 (INTAKE → GAP-ANALYSIS → CONTEXT-COMPLETION → VALIDATION) run automatically on every `@agent` invocation. Context loading must be invisible to the user and must not block agent startup.

**Impact:** CRITICAL — makes context loading automatic for every agent session.

---

## Problem Statement

**Current State:**
- Runtime engine: IMPLEMENTED (Story 5.3.1) at `.synapse/context-engine/engine.js`, `class ContextEngine`
- `.claude/hooks/agent-activation-tracker.cjs` already exists and runs on `UserPromptSubmit` (async, 3s timeout) — it detects the active agent (`@dev`, `@qa`, `@sm`, ...) and writes `session.active_agent` to `.synapse/metrics/hook-metrics.json`
- Engine Phases 1-4 are NOT wired to this (or any) hook — `ContextEngine` is implemented but never invoked automatically

**What's Missing:**
- A call from `agent-activation-tracker.cjs` into `ContextEngine` Phases 1-4 whenever an agent is detected
- Silent execution (no user prompts), with graceful timeout/degradation
- `session.context_state` populated alongside `session.active_agent` in `hook-metrics.json`

- [x] **AC2:** The call sequence uses the real `ContextEngine` API: `phase1_intake(userStatement)` → `phase2_gapAnalysis(intent)` → `phase3_contextCompletion(gaps)` → `phase4_validation(context)`, using the same method names/signatures as implemented in `.synapse/context-engine/engine.js` (Story 5.3.1)

## Technical Context

**Hook Lifecycle (existing, unchanged):**
- Event: `UserPromptSubmit` (registered in `.claude/settings.json`, currently `async: true`, `timeout: 3`)
- File: `.claude/hooks/agent-activation-tracker.cjs`
- Existing behavior: reads stdin (`{ message / user_message }`), runs `detectActiveAgent(prompt)`, writes `session.active_agent` to `.synapse/metrics/hook-metrics.json`, always responds `{}` on stdout
- Extension point: when `activeAgent` is truthy, additionally instantiate `ContextEngine` and run Phases 1-4 using the user prompt as the intake statement

**Engine Integration (real API — Story 5.3.1):**
```javascript
const ContextEngine = require('../../.synapse/context-engine/engine');

const engine = new ContextEngine();
const intent = await engine.phase1_intake(userStatement);
const gapAnalysis = await engine.phase2_gapAnalysis(intent);
const contextResult = await engine.phase3_contextCompletion(gapAnalysis.gaps);
const validation = await engine.phase4_validation(contextResult.context);
```

**Session State Update (extends existing `writeMetrics`):**
```yaml
session:
  active_agent:
    id: "dev"
    activated_at: "2026-06-10T10:00:00Z"
  context_state:
    phase_4_passed: true | false
    completed_at: "2026-06-10T10:00:00Z"
    intent_type: "..."
    completeness: 0.0-1.0
    gaps_detected: [...]
```

**Reference:** Story 5.3.1 engine implementation (`.synapse/context-engine/engine.js`)
