## Summary

Wire auto-contextualization engine into agent activation lifecycle. Create hook that automatically triggers Phase 1-4 (INTAKE → VALIDATION) when any agent is activated, making context loading invisible to the user. Hook must execute without blocking agent startup.

**Impact:** CRITICAL — Makes context loading automatic for every agent session.

---

**Current State:**
- Runtime engine: IMPLEMENTED (Story 5.3.1)
- Hook integration: NOT STARTED
- Activation trigger: None

**What's Missing:**
- Hook definition in `.claude/hooks/`
- Integration point in agent activation lifecycle
- Silent execution (no user prompts)
- Proper phase ordering (1-4 auto-run, 5-10 on-demand)

## Scope

### **IN (What We're Implementing)**
1. Create `agent-auto-contextualization.cjs` hook
2. Register hook in `.claude/settings.json`
3. Call engine Phases 1-4 synchronously
4. Populate `session.context_state` before agent greeting
5. Log to `.aiox/context-engine.log` (no console output)
6. Test hook execution on agent switch

### **OUT (What We're NOT Doing)**
- Phases 5-10 automatic execution (on-demand via tasks only)
- UI/prompts for context decisions
- Modify Constitutional gates (they run after hook)

---

## Technical Context

**Hook Lifecycle:**
- Event: `UserPromptSubmit` (fires after user input, before agent selection)
- Execution: Synchronous (blocking)
- Context available: `session`, `project`, `config`
- Timeout: 2 seconds max (degrade gracefully if exceeded)

**Engine Integration:**
```javascript
// Hook calls engine phases 1-4
const { runContextPhase } = require('.aiox-core/core/auto-contextualization');
const result = await runContextPhase(1, { intent: statement });
```

**Session State Update:**
```yaml
session:
  context_state:
    phase_4_passed: true | false
    completed_at: "2026-06-09T14:30:00Z"
    intent: "user statement"
    gaps_detected: [...]
    context_sources: { ... }
```
