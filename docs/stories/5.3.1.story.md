**Framework Reference:** `docs/auto-contextualization-engine.md`
- Phase 1: INTAKE — 1 statement, no parsing needed
- Phase 2: GAP-ANALYSIS — algorithm: matches intent against 8 knowledge sources
- Phase 3: CONTEXT-COMPLETION — sources: project registries, PROJECT.md, inference
- Phase 4: VALIDATION — 8-point completeness check
- Phase 5: IDS-CHECK — queries entity registry for patterns
- Phase 6: ROUTING — deterministic agent + workflow selection
- Phase 7: PRE-EXECUTION — loads templates from `.aiox-core/development/`
- Phase 8: EXECUTION — spawns selected agents
- Phase 9: HANDOFF — writes to `.aiox/handoffs/`
- Phase 10: PERSISTENCE — updates context registry

**Architecture Notes:**
- **Layer:** L4 (Project Runtime) — `.synapse/context-engine/` implements as project service, not framework core
- Zero user interaction (all phases silent)
- Async Phases 2-3 (gap analysis + context filling)
- Sync Phases 4-10 (validation → execution)
- Registry schema: implemented in Story 5.3.3
- Audit trail: `.aiox/gate-logs/` pattern
- **Integration:** Hooked via `agent-activation-tracker.cjs` (L3 hook) → triggers engine (L4 service)
