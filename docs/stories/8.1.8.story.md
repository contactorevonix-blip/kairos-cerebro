# Story 8.1.8: Documentation

**Status:** Ready
**Story ID:** 8.1.8
**Created:** 2026-06-11
**Epic:** EPIC-8 — Framework Evolution (Phase 1: Observability & Metrics)
**PRD Source:** `docs/prd/epic-8/phase-1-observability.md#818-documentation-1sp`
**Story Points:** 1

---

## Story

**As a** framework user (developer or maintainer),
**I want** complete documentation for the metrics CLI and the metrics system architecture,
**so that** I can install, query, export, and extend the observability system without reverse-engineering the implementation.

---

## Acceptance Criteria

1. File: `docs/guides/metrics-cli-guide.md`
   - Installation: `npm install` or `aiox install`
   - Command reference: `aiox metrics` all subcommands + examples
   - Query syntax: Filtering by article, date range, agent
   - Export examples: JSON, CSV for further analysis
2. File: `docs/architecture/metrics-system.md`
   - Architecture diagram: Data flow (logs → collector → DB → API → dashboard)
   - Schema reference: All metric types + units
   - Performance notes: Query timeouts, index strategy
   - Future extensibility: How to add new metric types
