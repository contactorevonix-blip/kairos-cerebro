## Description

Implement Constitutional Article III (Story-Driven Development) enforcement gate. Block git commits that don't reference an active story + ACs, or use `[no-story-req]` override for config-only commits.

**Article III Statement:** No code is written without a story associated. Every commit must trace to a story ID (e.g., `[Story 13.2]`) or use `[no-story-req]` override for exceptions.

## Acceptance Criteria

- [ ] Gate detects `git commit` at PreToolUse hook
- [ ] Parse commit message for story reference: regex `\[Story \d+\.\d+\]`
- [ ] If no story reference AND no `[no-story-req]` flag → BLOCK
- [ ] If `[no-story-req]` present → ALLOW but log as `override`
- [ ] Decision logged: article=`art-iii-story-driven`, reason shows commit msg + decision
- [ ] Test cases:
  - [ ] `git commit -m "feat: add X [Story 13.2]"` → ALLOWED
  - [ ] `git commit -m "fix: typo"` → BLOCKED (no story)
  - [ ] `git commit -m "chore: update config [no-story-req]"` → OVERRIDE logged
- [ ] Graceful degradation: if story file unreadable, downgrade to `warn-and-proceed`
- [ ] All 4 pre-commit gates PASS
