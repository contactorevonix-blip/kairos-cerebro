# Story 8.1.4: Dashboard Backend

**Status:** Ready
**Story ID:** 8.1.4
**Created:** 2026-06-11
**Epic:** EPIC-8 — Framework Evolution (Phase 1: Observability & Metrics)
**PRD Source:** `docs/prd/epic-8/phase-1-observability.md#814-dashboard-backend-2sp`
**Story Points:** 2

---

## Story

**As a** framework maintainer,
**I want** a REST API + minimal web dashboard backed by PostgreSQL,
**so that** I can visualize gate violations, story velocity, and metric trends without relying solely on the CLI.

---

## Acceptance Criteria

1. REST endpoints (CRUD metrics): `GET /api/metrics`, `POST /api/metrics/query`
2. Database connection: PostgreSQL with connection pooling
3. Web dashboard: `http://localhost:3000/metrics` (next.js static pages)
4. Authentication: API key (env var `METRICS_API_KEY`)
5. Performance: Dashboard loads <1s, queries <500ms
6. Charts: Gate violations timeline, story velocity, metric trends (min 5 charts)

**Non-Requirement:** Mobile-responsive design (desktop OK)

---

- **Depends on:** 8.1.1 (Metrics Schema Design), 8.1.3 (Gate Metrics Integration), 8.1.7 (Metrics Persistence — DB must be initialized)
