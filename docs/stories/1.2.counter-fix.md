## Story
**As a** founder do Kairos Check,
**I want** que o counter hero da landing page mostre dados reais ou seja honesto sobre o stage do produto,
**so that** developers técnicos que inspeccionem o código ou a API não percam confiança no produto.

## Acceptance Criteria

1. `/api/stats/counter` retorna dados reais (`verifyRequests` de `metrics.json`) quando `verifyRequests > 0`
2. Quando não há dados reais, o counter não mostra um número fabricado — mostra copy honesta (ex: "Beta" ou está oculto)
3. A função `counterBase()` é removida ou nunca usada como dado apresentado ao utilizador
4. O comportamento da proof bar (mostrar só quando `rawRequests > 0`) mantém-se inalterado — não regredir
5. A landing page continua a funcionar correctamente em todos os outros aspectos

## Dev Notes

**Ficheiros a modificar:**
- `packages/sniper-api/landing-page.js`
  - Linha 6-11: função `counterBase()` — fabricada (`180 + days * 400`)
  - Linha 13-22: `renderLandingPage()` — `rawRequests` e `rawBlocked` são dados reais, usados na proof bar
  - Linhas ~2386-2390: código JS cliente que chama `/api/stats/counter` e anima o counter hero
- `packages/sniper-api/server.js`
  - Linha 964-967: endpoint `/api/stats/counter` — actualmente retorna `counterBase()`

**Lógica actual (a substituir):**
```javascript
// landing-page.js
function counterBase() {
  const days = Math.floor((now - COUNTER_LAUNCH) / 86400000);
  return 180 + days * 400 + Math.floor(secs / 43); // ← fabricado
}
// server.js
{ count: counterBase() } // ← nunca são dados reais
```

**Lógica actual correcta a preservar (proof bar):**
```javascript
const rawRequests = m.verifyRequests || 0; // dados reais
const showProofBar = rawRequests > 0; // só mostra quando há dados — não regredir
```

**Regra do projecto:** "Nunca inventar dados — testimonials, contadores, actividade — só dados reais."

**Função `readGlobalMetrics()`** está disponível em `packages/sniper-api/server.js` — retorna os dados de `.kairos-data/metrics.json`.
