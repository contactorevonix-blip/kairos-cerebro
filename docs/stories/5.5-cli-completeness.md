# Story 5.5 — CLI Completeness

**Epic:** AIOX-OPS (AIOX Operacional 100%) | **Story ID:** 5.5 | **Effort:** 5sp | **Status:** Ready | **Executor:** @dev | **Quality Gate:** @qa

## Story

**As a** AIOX developer using the CLI,
**I want** 100% of registered tasks to be invocable via `aiox-tasks` and agent `*` commands to respond correctly,
**so that** there is zero gap between tasks registered in the registry and tasks actually reachable from the CLI.

## Goal

Verificar que: (1) `aiox-tasks.js` CLI cobre o registry de 213 tasks registadas, (2) comandos `*` de agente (ex: `*draft`, `*develop`, `*qa-gate`) respondem correctamente, e (3) os comandos CLI documentados em CLAUDE.md existem e funcionam. Produzir um relatório de coverage com gaps accionáveis.

## Contexto / Problema

GAP G7 (EPIC-AIOX-OPERACIONAL.md §3): `aiox-tasks.js` existe em `.claude/commands/`. A cobertura de `*tasks` e comandos de agente vs registry de 213 tasks não foi confirmada ponta-a-ponta. "Operacional" significa que todos os comandos declarados são invocáveis — não apenas os mais usados.

Adicionalmente, o AUDIT-PLAN.md § CLI não foi completamente executado: a verificação de gap entre tasks registadas e tasks invocáveis está pendente.

## Acceptance Criteria

- [ ] **AC1:** Inventário completo do task registry produzido — ler `.aiox/data/task-registry.json` (ou equivalente) e listar todas as tasks registadas com: `task_id`, `command`, `agent`, `status`.
  - *Given* o task registry, *When* lido, *Then* output lista N tasks (confirmar se N ≈ 213 conforme EPIC).

- [ ] **AC2:** Para cada task registada, verificar se é invocável via `aiox-tasks.js` CLI. Produzir resultado: `invocable: true/false`, `reason` (se false).
  - *Given* uma task registada, *When* `aiox-tasks list | grep <task_id>` corre, *Then* task aparece ou gap é documentado.

- [ ] **AC3:** Comandos `*` de agente verificados — pelo menos 1 comando por agente core (10 agentes × 1 comando = 10 verificações). Confirmar que respondem sem erro quando invocados via `/AIOX:agents:{agent}`.
  - *Given* `*draft` (sm), `*develop` (dev), `*qa-gate` (qa), *When* invocados, *Then* respondem correctamente.

- [ ] **AC4:** Comandos CLI documentados em CLAUDE.md verificados — `*create-story`, `*task {name}`, `*help`, `*exit` existem e respondem.
  - *Given* CLAUDE.md "Comandos com prefixo `*`", *When* verificados, *Then* cada um existe na definição do agente respectivo.

- [ ] **AC5:** Relatório `docs/qa/CLI-COVERAGE-REPORT.md` gerado com: total de tasks no registry, tasks invocáveis, tasks com gap, comandos de agente verificados, recomendações para gaps.
  - *Given* o relatório, *When* lido, *Then* lista explícita de gaps accionáveis (se existirem) com story de resolução sugerida.

- [ ] **AC6:** Output da Story 5.1 consultado — se a auditoria identificou gaps no `task-auto-suggest.cjs` hook (que sugere tasks), estes são incorporados no relatório.

## Scope

**IN:**
- Inventário do task registry (`.aiox/data/task-registry.json` ou equivalente)
- Verificação de coverage `aiox-tasks.js` vs registry
- Verificação de comandos `*` dos 10 agentes core
- Verificação de comandos CLI em CLAUDE.md
- Relatório `CLI-COVERAGE-REPORT.md`

**OUT:**
- Implementar tasks em falta (isso é scope de outra story se necessário)
- Modificar definições de agentes (L2 — extend-only)
- Modificar `.aiox-core/core/` (L1 — NEVER modify)
- Corrigir o `task-auto-suggest.cjs` hook (scope de 5.4 ou story separada se necessário)

## Dependencies

- **Prerequisito:** Story 5.1 (Hook Automation Audit) — confirma estado do `task-auto-suggest.cjs` hook que alimenta a CLI de tasks.
- **Bloqueia:** 5.7 (consolida outputs — inclui gaps de CLI).
- **Recursos:** `.aiox/data/task-registry.json`, `.claude/commands/aiox-tasks.js`, `.aiox-core/development/agents/*.md`, CLAUDE.md.

## Complexity Estimate

5sp — inventário do registry + verificação de coverage + relatório. Média complexidade: requer cross-reference entre registry, CLI e documentação. O número exacto de tasks pode variar (EPIC afirma 213).

## Business Value

Fecha o gap entre "declaramos que 213 tasks existem" e "confirmamos que 213 tasks são invocáveis". Qualquer comando documentado que não funciona é uma promessa quebrada para o utilizador — este story garante que o contrato é honrado.

## Risks

- **R1 (MEDIUM):** Registry pode não existir em `.aiox/data/task-registry.json` — pode estar noutro path. Mitigação: procurar `task-registry` em todo o repo antes de implementar AC1.
- **R2 (LOW):** 213 tasks pode ser um número exagerado — registry real pode ter menos. Mitigação: contar tasks reais e documentar discrepância vs EPIC.
