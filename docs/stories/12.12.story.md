### Squad Orchestration Workflow

@squad-creator follows squad creation task:
1. **Define Squad Goal:** What is the squad trying to accomplish?
2. **Compose Agents:** Which agents are needed? (core: @dev, @qa, @architect; specialists: @data-engineer, @ux, etc.)
3. **Map Workflows:** Define how agents coordinate (handoff sequence, parallel tasks)
4. **Create squad.yaml:** Encode squad definition in YAML
5. **Register Agents:** Configure each agent for squad mode (shared context, memory)
6. **Execute Workflow:** Run squad through SDC or other workflows
7. **Clone DNA:** Export squad expertise (used to train new squads)

### Research Alignment

**Orchestrator-Worker Pattern** (research rec #2):
- @squad-creator is meta-orchestrator (above individual agent orchestration)
- Squads are composed of agents
- Multi-agent parallelization enabled

**Task-First Workflow** (research rec #6):
- Squad executes tasks (not agents deciding independently)
- Task coordination explicit in squad.yaml
