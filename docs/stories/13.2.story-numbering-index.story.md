# Story 13.2: Story Numbering Authority Index

**Epic:** EPIC-13

**Status:** Ready

## Story Statement

**As a** Scrum Master,  
**I want** a single authoritative index of all story locations and numbering schemes,  
**so that** I can reliably check if a story exists without ambiguity.

---

## Problem

Stories exist in **3 naming schemes**:
1. **Flat:** `docs/stories/1/1.1.story.md`, `docs/stories/1/1.18.story.md`
2. **Epic-nested:** `docs/stories/epics/epic-5-governance/5.2-WORKFLOW-DESIGN.md`
3. **Orphaned root:** `docs/stories/1.8-aiox-compliance-audit.md` (not in any folder)

**Risk:** @sm cannot reliably check if story "1.8" exists. It might be in folder `1/1.8.story.md` OR as root file `1.8-*.md` OR in an epic subfolder.

**Impact:** Duplicate stories created, orphaned stories forgotten, story numbering confusion.

## Technical Context

### Current Story Structure

**Schema 1 (Flat):**
- Pattern: `docs/stories/{epic_num}/{epic_num}.{story_num}.story.md`
- Example: `docs/stories/1/1.1.story.md`, `docs/stories/1/1.18.story.md`
- Regex: `^docs/stories/\d+/\d+\.\d+\.story\.md$`

**Schema 2 (Epic-nested):**
- Pattern: `docs/stories/epics/{epic_folder}/{story_num}-{title}.md`
- Example: `docs/stories/epics/epic-5-governance/5.2-WORKFLOW-DESIGN.md`
- Regex: `^docs/stories/epics/[^/]+/\d+\.\d+-.*\.md$`

**Schema 3 (Orphaned root):**
- Pattern: `docs/stories/{epic_num}.{story_num}-{title}.md`
- Example: `docs/stories/1.8-aiox-compliance-audit.md`
- Regex: `^docs/stories/\d+\.\d+-.*\.md$`

### INDEX Format

```markdown
# Story Index — Authoritative Master List

## Schema Legend
- **Flat:** `docs/stories/{epic}/{epic}.{story}.story.md`
- **Epic-nested:** `docs/stories/epics/{epic-folder}/{story}-title.md`
- **Archive:** `docs/stories/.archived/{story}-title.md`

## All Stories (Organized by Epic)

### Epic 1: AIOX Architecture Foundation
- 1.1: file path | status
- 1.2: file path | status
- 1.3: file path | status

...and so on
```

## Tasks / Subtasks

1. **Audit All Story Files**
   - Scan `docs/stories/` recursively
   - List all files with paths
   - Classify each by schema (flat/nested/orphaned)
   - Document in decision log
   - Link: AC1

2. **Create INDEX-AUTHORITATIVE.md**
   - Start with template above
   - Add all stories found in audit
   - Group by epic number
   - Include file path, status (Draft/Ready/InProgress/Done), and schema type
   - Link: AC1, AC6

3. **Document Regex Patterns**
   - Add "Schema Legend" section to INDEX
   - Write regex for each of 3 schemas
   - Add examples
   - Link: AC2

4. **Identify Orphaned Stories**
   - Find all stories in `docs/stories/` root (not in `1/`, `epics/`, etc.)
   - For each: determine correct epic folder
   - Mark in INDEX for reclassification
   - Link: AC3

5. **Create Pre-Commit Validation Script**
   - Script: `validate-story-index.cjs`
   - Logic:
     1. Read INDEX-AUTHORITATIVE.md
     2. Parse story list
     3. For each story, check file exists
     4. Report missing files
     5. Report files not in INDEX
   - Fail if drift found
   - Link: AC4

6. **Add Pre-Commit Hook**
   - Update `.husky/pre-commit` to call `npm run validate-story-index`
   - Or create `.claude/hooks/story-index-validator.cjs` (if using Claude hooks)
   - Link: AC4

7. **Update @sm Instructions**
   - Document in `.claude/rules/` or SM agent skill
   - Add: "Before creating story X.Y, check INDEX-AUTHORITATIVE.md for conflicts"
   - Link: AC5

8. **Move Orphaned Stories**
   - Based on audit (Task 4), move orphaned stories to proper epic folders
   - Update INDEX
   - Create archive folder for deprecated stories (if any)
   - Link: AC3
