## Story

**Title:** Fix Enforcement Gates YAML Regex — Framework Protection Detection

**Epic:** EPIC-13 — AIOX Integration Auditoria Foundation

**Summary:**
Regex for detecting `boundary.frameworkProtection: false` in `core-config.yaml` is too restrictive and fails with natural YAML formatting. Fix regex to properly handle multi-line YAML with Windows/Unix line endings.

**Problem Statement:**
The `isFrameworkProtectionEnabled()` function in `.claude/hooks/enforce-quality-gates.cjs` uses a regex that requires exact line break pattern: `/boundary:\s*[\r\n]+\s*frameworkProtection:\s*false/i`

This fails when:
- YAML formatting includes spaces/comments between lines
- Single newline instead of multiple
- Mixed line endings in configuration

**Impact:**
- Framework boundary protection gate (Art. VI-VII) may not correctly detect disabled protection
- False positives/negatives in enforcement

## Technical Notes

**Regex Analysis:**

Broken: `[\r\n]+` — requires one or more newlines exactly at that position
Fixed: `[^]*?` — matches any character (including newlines), non-greedy

**Line Ending Handling:**
- Windows: `\r\n` (CRLF)
- Unix: `\n` (LF)
- Both handled by `[^]*?` without explicit escaping

**Word Boundary:**
- Added `\b` at end to prevent matching `frameworkProtection: false_flag`
