## Story

**As a** framework architect,
**I want** a module that extracts decision frameworks and workflow logic from mentor agents into a state-machine model,
**so that** cloned squads inherit the mentor's reasoning patterns and command execution order.

---

## Acceptance Criteria

- [x] **AC1:** Module `squads/squad-creator/core/thinking-dna.js` created with `extractThinkingDNA(agentYaml)` function
- [x] **AC2:** DNA model includes: `decision_framework` (trigger/options/default pattern), `workflow_chains` (command → task sequence), `error_recovery` (error_type → action mapping)
- [x] **AC3:** DNA output stored as JSON to `.aiox/squad-dnas/{agent-id}-thinking.json` (via `saveThinkingDNA`)
- [x] **AC4:** Unit tests verify 90%+ workflow alignment: `validation.alignment` = fraction of mapped tasks resolving to real files; test asserts `>= 0.9` (1.0 for @dev)
- [x] **AC5:** Workflow chains map to real tasks in `.aiox-core/development/tasks/` (existence-verified; missing tasks reported, never invented; inline `# comments` stripped)

---

## Dependencies

- **Depends on:** None — independent from 8.3.1
- **Blocks:** 8.3.3 (Squad Template Generation), 8.3.8 (Integration Tests)

---

## Scope

**IN:**
- Commands array parsing (visibility, name, description)
- Task dependency extraction from commands
- Decision tree inference from task ordering
- Error handling pattern detection

**OUT:**
- Runtime behavior prediction (that's Phase 4 validation)
- Machine learning on agent patterns
- Live decision tracing

---

## Risks

- **Risk:** Extracted chains don't match actual execution (hidden complexity)
- **Mitigation:** Validate against real task files; test with mentor agent before cloning
