**Epic:** PHASE 1 — Constitutional Enforcement

## Goal

Configurar e ativar AIOX completo, sincronizar toda a framework na pasta de trabalho e executar todas as validações de compliance (Constitution gates, règras de deny/allow, hooks, scores).

## Business Value

Garante que o AIOX está 100% funcional e compliant com a Constitution, permitindo que todas as ferramentas de automação (hooks, rules, validation gates) funcionem corretamente em desenvolvimento.

## Prerequisites

- ✅ Story 1.4 — Hooks Setup (DONE)
- ✅ Story 1.5 — Deny Rules Configuration (DONE)
- ✅ Story 1.6 — Validation Gates (DONE)

*Nota: Estas 3 stories devem estar 100% completas antes de iniciar 1.7.*

## Scope

### In Scope
- Setup completo da CLI AIOX
- Configuração de `core-config.yaml` e `.claude/settings.json`
- Integração de hooks Git (pre-commit, pre-push)
- Validação de todos os 6 artigos da Constitution
- Validações de qualidade (lint, typecheck, test, build)
- Geração de relatório final de status

### Out of Scope
- Criação de novas features ou agentes
- Modificação de `.aiox-core/core/` ou `.aiox-core/development/` (L1/L2 framework)
- Deploy para produção
- Treino de novos utilizadores

## Risks

1. **Hook Integration Failures** — Permissões incorretas podem bloquear commits válidos
   - *Mitigation:* Testar manualmente cada hook após instalação

2. **Configuration Conflicts** — Diferentes valores em `core-config.yaml` vs `.claude/settings.json`
   - *Mitigation:* Validar schema antes de aplicar

3. **Dependency Circular Detection** — Graph generation pode falhar se há ciclos não-detectados
   - *Mitigation:* Executar `aiox graph` e revisar manualmente

4. **Quality Gate Failures** — Code coverage ou syntax checks podem falhar em commits antigos
   - *Mitigation:* Documentar baseline, não bloquear por isto

## Technical Notes

**AIOX Configuration Stack:**
- `bin/aiox.js` — CLI entry point
- `.aiox-core/constitution.md` — regras não-negociáveis
- `.aiox-core/data/` — core config files
- `core-config.yaml` — feature flags + settings
- `.claude/settings.json` — permission + deny/allow rules
- `.husky/` — git hooks infrastructure

**Validation Commands:**
```bash
aiox doctor
aiox graph --deps --format=json
aiox validate-constitution
npm run lint
npm run typecheck
npm test
npm run build
```

**Deny/Allow Rules Files:**
- `.claude/settings.json` — local project rules
- `.claude/rules/agent-authority.md` — agent delegation matrix
