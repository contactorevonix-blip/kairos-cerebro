# Story 13.3: Load Workflow Definitions

**Story ID:** 13.3  
**Epic:** EPIC-13 (Full Context Determinism)  
**Status:** Ready  
**Points:** 5sp  
**Owner:** @dev  
**Dependencies:** 13.1

## Description

Load all 4 workflow definitions from `.aiox-core/development/workflows/*` with task sequence validation. Workflows activate agents through orchestrated task sequences that must be deterministic and validated before execution.

## Acceptance Criteria

- [ ] Loads all 4 workflow definitions from `.aiox-core/development/workflows/*`
- [ ] Parses workflow structure per L3 schema (see Technical Notes)
- [ ] Validates task sequence DAG (no circular dependencies, topological sort)
- [ ] Caches workflow definitions in `.synapse/workflow-cache/{workflow-id}.json` per schema
- [ ] Cache TTL: 1 hour (configurable via `.aiox-core/data/cache-config.yaml`)
- [ ] On workflow activation, loads cache if valid, else reloads from disk
- [ ] Logs cache hit/miss + validation results to `.aiox/context-load-logs/{date}.jsonl`
- [ ] All 4 pre-commit gates PASS (lint ✅, typecheck ✅, test ✅, CodeRabbit pending)

## Technical Notes

**L3 Schema for Workflow Definition (`.synapse/workflow-cache/{workflow-id}.json`):**

```json
{
  "id": "workflow-sdc",
  "name": "Story Development Cycle",
  "version": "1.0.0",
  "description": "Primary development workflow: @sm→@po→@dev→@qa→@devops",
  "phases": [
    {
      "phase": 1,
      "name": "Create",
      "agent": "@sm",
      "tasks": ["*draft"],
      "transitions": { "success": "phase_2" }
    },
    {
      "phase": 2,
      "name": "Validate",
      "agent": "@po",
      "tasks": ["*validate-story-draft"],
      "transitions": { "pass": "phase_3", "fail": "phase_1" }
    },
    {
      "phase": 3,
      "name": "Implement",
      "agent": "@dev",
      "tasks": ["*develop-story"],
      "transitions": { "success": "phase_4" }
    },
    {
      "phase": 4,
      "name": "QA Gate",
      "agent": "@qa",
      "tasks": ["*qa-gate"],
      "transitions": { "pass": "phase_5", "fail": "phase_3" }
    },
    {
      "phase": 5,
      "name": "Push",
      "agent": "@devops",
      "tasks": ["git push"],
      "transitions": { "success": "complete" }
    }
  ],
  "dependencies": {
    "phase_1": [],
    "phase_2": ["phase_1"],
    "phase_3": ["phase_2"],
    "phase_4": ["phase_3"],
    "phase_5": ["phase_4"]
  },
  "metadata": {
    "created_at": "2026-06-17T00:00:00Z",
    "modified_at": "2026-06-18T00:00:00Z",
    "determinism_score": 100,
    "is_canonical": true
  }
}
```

**Validation Requirements:**
- Each phase must have exactly 1 agent
- Tasks must be non-empty array
- Transitions must map to valid phases (or "complete")
- Dependency graph must be a DAG (validate using topological sort)
- No circular dependencies allowed

**Implementation Pattern (story 13.4 will handle Kahn sort):**
- Story 13.3: Load + parse + validate DAG structure
- Story 13.4: Topological sort (Kahn algorithm) for optimal execution order
