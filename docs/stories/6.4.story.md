## Summary

Align agent handoff protocol between documentation (`.claude/rules/agent-handoff.md`) and actual implementation (`.aiox/handoffs/` files). Currently documented as YAML compaction protocol (with `from_agent`, `to_agent`, `consumed` fields, 500-token budget) but implemented as JSON commit logs (different schema). 

**Architecture Decision Required:** Update docs to match implementation OR implement documented protocol. This story is primarily an architecture decision + implementation of the chosen path.

**Impact:** CRITICAL â€” Gap blocks handoff consolidation rule (Story 6.5) and creates cognitive dissonance for developers.

## Problem Statement

**Current State (Documented):**
- `.claude/rules/agent-handoff.md` describes YAML compaction protocol
- Filename pattern: `handoff-{from}-to-{to}-{timestamp}.yaml`
- Fields: `from_agent`, `to_agent`, `story_context`, `decisions`, `files_modified`, `blockers`, `next_action`, `consumed: bool`
- Purpose: Compact agent context (~379 tokens) on agent switches

**Current State (Actual Implementation):**
- `.aiox/handoffs/` contains JSON files (not YAML)
- Filename pattern: `handoff-{timestamp}-{story_id}.json`
- Fields: `timestamp`, `story_id`, `story_path`, `story_status`, `agent`, `action`, `notes`
- No `from_agent`/`to_agent` fields, no `consumed` flag
- Schema matches `.aiox/task-logs/` structure (commit logs, not handoff compaction)

**Open Questions:**
1. Is documented YAML protocol aspirational (planned but not built)?
2. Is JSON format the "real" implementation and docs are outdated?
3. Is agent handoff compaction implemented elsewhere?

**What's Missing:**
- One source of truth for handoff protocol

## Technical Context

**Documented Protocol (agent-handoff.md):**
```yaml
handoff:
  from_agent: "{agente A}"
  to_agent: "{agente B}"
  story_context:
    story_id: "..."
    story_path: "..."
    story_status: "..."
    current_task: "..."
    branch: "..."
  decisions:
    - "..."
  files_modified: [...]
  blockers: [...]
  next_action: "..."
  consumed: false
```

**Actual Implementation (JSON files):**
```json
{
  "timestamp": "2026-06-09T09:32:26.201Z",
  "story_id": "unknown",
  "story_path": "docs/stories/...",
  "story_status": "InProgress",
  "agent": "dev",
  "action": "commit",
  "notes": "Story... committed with status: ..."
}
```

**Decision Options:**
1. **Keep YAML (Documented):** More structured, compacts context, supports `consumed` flag for memory management
2. **Keep JSON (Current):** Simpler schema, matches task-logs pattern, less overhead
