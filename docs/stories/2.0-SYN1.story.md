## Technical Notes

### Hook Chain Architecture

**Hook Execution Points:**
- `UserPromptSubmit`: fires AFTER user input, BEFORE agent selection
- `PreToolUse`: fires BEFORE tool execution
- `PostToolUse`: fires AFTER tool execution
- `AgentActivation` (custom): fires when `@agent` is invoked

**Current Hook Configuration (.claude/settings.json):**
```json
{
  "hooks": {
    "UserPromptSubmit": [...existing hooks...],
    "AgentActivation": [...should include agent-activation-tracker.cjs...]
  }
}
```

**Previous Failure (Deleted Hook):**
- Hook: `agent-activation-tracker.cjs`
- Reason for failure: `session` object não estava available na context
- Lesson: Hook precisa ser invocado APÓS agent selection, não antes

### Session State Model

**Expected Structure:**
```javascript
{
  "session": {
    "id": "session-123-456",
    "active_agent": {
      "id": "sm",          // ← MUST be populated
      "name": "River",
      "persona": "Scrum Master",
      "invoked_at": "2026-06-08T10:15:23Z"
    },
    "timestamp": "2026-06-08T10:15:23Z"
  }
}
```

**Current Problem:**
```javascript
{
  "session": {
    "id": "...",
    "active_agent": null  // ← ALWAYS null — this is the bug
  }
}
```

### SYNAPSE Layers 2-7 Dependencies

| Layer | Rule Category | Status | Depends On |
|-------|---------------|--------|-----------|
| L2 | Agent Authority | BLOCKED | `session.active_agent.id ≠ null` |
| L3 | Story Lifecycle | BLOCKED | `session.active_agent.id ≠ null` |
| L4 | Workflow Execution | BLOCKED | `session.active_agent.id ≠ null` |
| L5 | Code Intelligence | BLOCKED | `session.active_agent.id ≠ null` |
| L6 | Framework Boundary | BLOCKED | `session.active_agent.id ≠ null` |
| L7 | IDS Principles | BLOCKED | `session.active_agent.id ≠ null` |

Once `session.active_agent.id` is populated, ALL L2+ rules should load automatically via SYNAPSE rule-loader.
