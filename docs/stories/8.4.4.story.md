# Story 8.4.4: Automated Blocker Resolution

**Status:** InReview
**Story ID:** 8.4.4
**Created:** 2026-06-11
**Epic:** EPIC-8 — Framework Evolution (Phase 4: Auto-Healing)
**PRD Source:** `docs/prd/epic-8/phase-4-auto-healing.md#844-automated-blocker-resolution-15sp`
**Story Points:** 1.5

---

## Story

**As a** development lead,
**I want** the system to detect story blockers and suggest workarounds or parallel tracks,
**so that** teams don't get stuck waiting for upstream stories and can find alternative paths forward.

---

## Acceptance Criteria

- [x] **AC1:** Module `.aiox-core/core/auto-heal/blocker-resolver.js` created
- [x] **AC2:** Detection patterns:
  - Dependency story not DONE → suggest parallel track
  - Circular dependency (A blocks B, B blocks A) → suggest story split
  - Resource unavailable (PR approval pending) → suggest timeout + escalation
- [x] **AC3:** CLI command `aiox blocker check {storyId}` returns:
  - Blocker status (description + blocker story ID)
  - Estimated resolution time (days until dependency completes)
  - Suggested workarounds (3-5 options ranked by feasibility)
- [x] **AC4:** Developer can accept workaround or escalate to @aiox-master
- [x] **AC5:** All blocker resolutions logged to `.aiox/blocker-resolutions.jsonl` (audit trail)

---

## Dependencies

- **Depends on:** Story tracking system (Phase 1 metrics)
- **Blocks:** None — final Phase 4 story

---

## Scope

**IN:**
- Story dependency graph traversal
- Blocker pattern matching
- Workaround generation
- Escalation routing

**OUT:**
- Automatic blocker fixing (that would break dependencies)
- ML-based ETA prediction

---

## Risks

- **Risk:** Suggestions inappropriate (workaround breaks other things)
- **Mitigation:** Suggestions are advisory; developers must validate
