# Story 1.4 — 30-Day Archival Job

**Epic:** EPIC-1 (Monitor Core)  
**Status:** Draft  
**Complexity:** 8sp  
**Assignee:** @dev  
**Created:** 2026-06-04

---

## Summary

Implement background archival job that runs daily at 01:00 UTC. Compresses audit logs and SQLite snapshots older than 30 days into `tar.gz` files and moves them to `.aiox/archives/`. Maintains a rolling 30-day retention window to keep disk usage <1GB per 30d (NFR3).

**Why:** Monitor must retain 30 days of history without unbounded disk growth. Archival keeps recent data hot (fast queries) and old data compressed (cheap storage).

---

## Acceptance Criteria

- [ ] Create `src/archival/ArchivalJob.ts`:
  - Runs on schedule: daily at 01:00 UTC
  - Identifies files older than 30 days (audit logs, SQLite backups)
  - Compresses to `archive-{YYYY-MM-DD}-{hash}.tar.gz` (max 100MB per archive)
  - Moves to `.aiox/archives/{year}/{month}/`
  - Updates `.aiox/ARCHIVAL-INDEX.json` with manifest

- [ ] Archival index schema:
  ```typescript
  interface ArchivalIndex {
    archives: {
      filename: string;
      created_at: ISO8601;
      size_bytes: number;
      file_count: number;
      date_range: { start: ISO8601; end: ISO8601 };
      checksum_sha256: string;
    }[];
    total_archives: number;
    last_run: ISO8601;
  }
  ```

- [ ] Disk usage guarantee: <1GB for any 30-day window
- [ ] Deletion: files older than 30 days are deleted from hot storage after archival
- [ ] Archive integrity: checksum verification after compression
- [ ] Retention policy: archives kept for 90 days (cold storage), then deleted
- [ ] Audit log: archival job logs itself (AuditLog entry on start/completion)
- [ ] Tests: simulate date progression, verify archival triggers correctly

---

## Implementation Notes

- **Scheduler:** Use node-schedule or Bull (check existing stack)
- **Compression:** tar + gzip (standard, no external deps)
- **Constitution:** Art. I (CLI first) — archival is backend-only operation
- **No deletion during window:** only compress, only delete after 30 days
- **Idempotency:** can be run multiple times safely
