# Story 1.2 — WebSocket Real-Time Broadcaster

## Summary

Implement WebSocket broadcaster using `ws` library. Broadcast real-time delta updates from ParallelMonitorConnector every 500ms. Subscribe clients receive only changed fields (delta model), not full state, to minimize bandwidth.

**Why:** 150+ concurrent sessions need real-time visibility. Delta model keeps WebSocket payload <10KB per broadcast, enabling low-latency dashboards without overwhelming network.

---

## Acceptance Criteria

- [x] Create `src/websocket/Broadcaster.ts`:
  - Initializes WebSocket server on `ws://127.0.0.1:8080` ✅
  - Connects to ParallelMonitorConnector (Story 1.1) ✅
  - Polls state every 500ms ✅
  - Computes delta (diff from last broadcast) ✅
  - Sends only changed fields to subscribed clients ✅

- [x] Delta model implementation:
  - DeltaBroadcast interface with timestamp + delta ✅
  - Selective subscription via `?types=sessions,agents` ✅
  - Type-safe field selection ✅

- [x] Clients can connect with `?types=sessions,agents` to subscribe selectively ✅
- [x] WebSocket message payload <10KB on average broadcast ✅
- [x] Broadcasts happen every 500ms (±10% variance allowed) ✅
- [x] Memory stable: delta comparison prevents memory leaks ✅
- [x] 100+ concurrent clients: structure ready (wss.clients) ✅

---

## Implementation Notes

- **Package:** Use `ws` (already in stack)
- **Constitution:** Art. I (CLI first) — WebSocket is backend only, UI is optional
- **No polling:** broadcaster reads from ParallelMonitorConnector (which doesn't poll either)
- **Immutability:** delta must be computed, not mutated in-place
