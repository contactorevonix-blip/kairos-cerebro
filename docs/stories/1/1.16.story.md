## Summary

Implement auto-enforcement of AIOX Constitution articles II-VII. Create validation gates that automatically detect and block violations of:
- Art. II: Agent Authority (only @devops can git push)
- Art. III: Story-Driven Development (no code without story)
- Art. IV: No Invention (specs validated against requirements)
- Art. V: Quality First (gates block merge if criteria not met)
- Art. VI-VII: Additional governance rules

**Why:** SYNAPSE L2+ layers are now loading, but Constitutional enforcement is not active. Violations pass through silently. This story operationalizes the Constitution via automated gates in hooks.

---

## Acceptance Criteria

- [ ] AC1: Art. II (Agent Authority) enforced
  - Hook detects git push from non-@devops agent
  - Blocks with error: "Only @devops can execute git push"
  - Exception: Allow if user explicitly overrides with `--skip-devops-check`

- [ ] AC2: Art. III (Story-Driven) enforced
  - Hook detects code commit without associated story validation
  - Blocks with error: "Code requires story validation (status: Ready or higher)"
  - Allows commits with `[no-story-req]` tag (for config-only changes)

- [ ] AC3: Art. IV (No Invention) enforced
  - Validates spec.md statements against requirements
  - Blocks specs with invented features (not in FR, NFR, or research findings)
  - Warning log for soft violations

- [ ] AC4: Arts. V-VII enforced
  - Quality gates block merge if quality criteria not met
  - Framework boundary protection (L1/L2 protected from modification)
  - Additional governance rules active

- [ ] AC5: Enforcement metrics
  - `.synapse/metrics/hook-metrics.json` tracks:
    - `gatesEnforced`: count of active gates
    - `violationsDetected`: count of detected violations
    - `violationsBlocked`: count of blocked violations
    - `overridesUsed`: count of explicit overrides

- [ ] AC6: Documentation
  - `.claude/rules/enforcement-gates.md` documents all 7 articles
  - Override policy documented (when allowed, how to use `--skip-*`)
  - Gate decision logs available in `.aiox/gate-logs/`

## Implementation Notes

- **Gate Architecture:** Pre-commit and pre-push hooks intercept violations
- **Constitution:** Reference `.aiox-core/constitution.md` (Art. I-VII)
- **Layer Context:** Use L2 Agent Layer + L3 Workflow Layer for context
- **Graceful Degradation:** Gate timeout doesn't block development (warn-and-proceed)
- **Idempotency:** Gates can run multiple times without side effects
