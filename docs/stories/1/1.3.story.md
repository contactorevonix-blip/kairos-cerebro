## Summary

Design and implement immutable audit log to record all Monitor + Control events (session start/stop, agent activation, story validation, constitutional gates, rate limit hits, archival jobs). Log is append-only JSON-L (one entry per line), never mutated, never deleted. Serves as forensic source of truth for the 30-day rolling window.

**Why:** NFR5 in PRD requires immutable audit log. Needed for compliance, debugging, and detecting anomalies. Constitution (Art. II) enforcement must be audited.

## Implementation Notes

- **No database:** audit log is file-based, not SQLite (more immutable)
- **Write-through:** every event hits disk immediately (no buffer queue)
- **Constitution:** Art. II requires auditability of gate enforcement — this is the mechanism
- **Rotation strategy:** daily file, yesterday's file becomes read-only (archived)
