## Summary

Design and implement immutable audit log to record all Monitor + Control events (session start/stop, agent activation, story validation, constitutional gates, rate limit hits, archival jobs). Log is append-only JSON-L (one entry per line), never mutated, never deleted. Serves as forensic source of truth for the 30-day rolling window.

**Why:** NFR5 in PRD requires immutable audit log. Needed for compliance, debugging, and detecting anomalies. Constitution (Art. II) enforcement must be audited.

---

## Acceptance Criteria

- [ ] Create `src/audit/AuditLog.ts`:
  - Appends to `kairos-monitor-audit-{YYYY-MM-DD}.jsonl` file
  - Never overwrites or mutates existing entries
  - Rotates file daily at 00:00 UTC
  - No in-memory buffering (write immediately, sync to disk)

- [ ] Audit log entry schema:
  ```typescript
  interface AuditLogEntry {
    timestamp: ISO8601;
    event_type: 'session_start' | 'session_stop' | 'agent_activated' | 
                'story_validated' | 'gate_enforcement' | 'rate_limit_hit' |
                'archival_triggered' | 'archival_completed';
    details: {
      session_id?: string;
      agent_id?: string;
      story_id?: string;
      gate_type?: string;  // e.g., 'git_push', 'mcp_add', 'gh_pr'
      reason?: string;
      actor?: string;      // user, hook, agent
    };
    sequence: number;      // monotonically increasing
  }
  ```

- [ ] API: `logEvent(type, details)` — async, returns after disk sync
- [ ] Immutability guarantee: file permissions 444 (read-only) after rotation
- [ ] Daily rotation with compression: `kairos-monitor-audit-{YYYY-MM-DD}.jsonl.gz` (moved to `.aiox/audits/`)
- [ ] Query API: `readAuditLog(startDate, endDate)` — returns all entries in range
- [ ] Performance: single event logged in <10ms (disk I/O latency)
- [ ] Tests verify: no data loss, correct daily rotation, immutability

---

## Implementation Notes

- **No database:** audit log is file-based, not SQLite (more immutable)
- **Write-through:** every event hits disk immediately (no buffer queue)
- **Constitution:** Art. II requires auditability of gate enforcement — this is the mechanism
- **Rotation strategy:** daily file, yesterday's file becomes read-only (archived)
