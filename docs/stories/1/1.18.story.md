# Story 1.18 — Rastreabilidade Synchronization

## Summary

Implement file tracking and entity registry for complete rastreabilidade (traceability). Enable tracing from requirement → task → code → test. Create bidirectional mapping:
- Requirement (FR/NFR) → Stories → Tasks → Files → Tests
- File → Story → Epic → Requirement → Success Metrics

Maintain Entity Registry (`.aiox-core/data/entity-registry.yaml`) that tracks all artifacts: stories, tasks, templates, rules, agents, hooks, and their relationships.

**Why:** System has no way to know what code implements what requirement. Files are created ad-hoc, untracked. Entity Registry exists but is never synchronized. Rastreabilidade is zero. This story operationalizes the traceability chain.

---

## Acceptance Criteria

- [ ] AC1: File tracking system
  - Hook intercepts Write, Edit, Delete operations
  - Logs each file change with context:
    - File path, operation (create/update/delete)
    - Story ID (if in story context)
    - Decision context (why this file)
    - Timestamp + agent (who made the change)
  - Persisted to `.aiox/file-changes.jsonl` (append-only log)

- [ ] AC2: Entity registry synchronized
  - Registry (`.aiox-core/data/entity-registry.yaml`) tracks all artifacts:
    - Stories (ID, status, effort, AC count, files created)
    - Tasks (ID, category, trigger_context, dependencies)
    - Templates (ID, type, usage count)
    - Rules (ID, article, enforced)
    - Agents (ID, persona, commands)
    - Hooks (ID, type, events)
  - Registry updated on every Write/Edit (via hook)
  - No manual entry required

- [ ] AC3: Rastreabilidade tracing
  - Enable backward trace: File → Story → Epic → Requirement
  - Enable forward trace: Requirement → Story → Task → File → Test
  - CLI command: `aiox trace requirement {id}` (show all downstream)
  - CLI command: `aiox trace file {path}` (show all upstream)

- [ ] AC4: Synchronization validation
  - Registry state always matches git state (files exist + are tracked)
  - No orphaned files (files not in any story)
  - No orphaned stories (stories not in any epic)
  - Validation gate: CI/CD blocks merge if registry out-of-sync

- [ ] AC5: Metrics + reporting
  - `.synapse/metrics/hook-metrics.json` tracks:
    - `filesTracked`: total files in system
    - `entitiesTracked`: total artifacts in registry
    - `rastreabilidadeCoverage`: % of files with traceability
    - `lastRegistrySyncTime`: timestamp of last sync
    - `syncValidationPassed`: true/false

- [ ] AC6: Documentation
  - `.claude/rules/rastreabilidade.md` documents:
    - File tracking flow (when files are logged)
    - Entity registry format (schema + examples)
    - Traceability protocol (how to trace)
    - Registry sync process (automatic + manual override)

## Implementation Notes

- **File Hook:** PostToolUse hook (after Write/Edit completes)
- **Registry Format:** YAML with relationships (`usedBy:`, `requires:`, `implements:`)
- **Append-Only Log:** File changes logged to JSONL (immutable audit trail)
- **Sync Validation:** Compare registry against git state (no blocking on failure)
- **CLI Integration:** Add `aiox trace` commands for traceability queries

   - Create `.aiox-core/core/entity-registry/sync-registry.js`

   - Scan `.aiox-core/` for all artifacts

   - Validate registry state matches git state

4. **Add rastreabilidade tracing** (AC3)
   - Create CLI command: `aiox trace requirement {id}` (query registry)

   - Allow override with `--skip-registry-check`

6. **Add metrics + reporting** (AC5-AC6)
   - Update `.synapse/metrics/hook-metrics.json` schema
   - Calculate rastreabilidade coverage (% of files traced)
   - Track sync validation results
   - Create `.claude/rules/rastreabilidade.md`

7. **Test rastreabilidade scenarios** (all ACs)
   - Test: Create file → logged to file-changes.jsonl
   - Test: Registry sync detects new story → adds to registry
   - Test: Trace requirement → returns story + files + tests
   - Test: Trace file → returns requirement + story + task
   - Test: Validation gate blocks push if registry out-of-sync
   - Test: Metrics reflect tracking activity

## Dev Notes

**File Change Log Format (JSONL):**
```json
{
  "timestamp": "2026-06-08T15:32:00Z",
  "operation": "create|update|delete",
  "file_path": "src/components/Button.tsx",
  "story_id": "1.16",
  "agent_id": "dev",
  "context": "AC1: Implement Button component",
  "size_bytes": 2048
}
```

**Entity Registry Relationships:**
- `usedBy`: [list of stories/tasks that use this artifact]
- `requires`: [list of artifact dependencies]
- `implements`: [list of requirements this artifact implements]
- `relatedTo`: [list of related artifacts]

**Registry Sync Process:**
1. Load entity-registry.yaml
2. Scan `.aiox-core/` for artifacts (tasks, templates, rules, agents, hooks)
3. Scan `docs/stories/` for stories + epics
4. Compare against registry
5. Add missing entries, mark stale entries
6. Validate no orphaned entities
7. Save registry with sync timestamp
