## Summary

Build React dashboard skeleton using Vite + TypeScript. Connect to Monitor WebSocket (`ws://127.0.0.1:8080/monitor`). Implement basic layout with 4 main sections: Sessions, Agents, Tasks, Hooks. All sections empty initially (data populated by stories 5.2-5.4). Dashboard is read-only observation layer (Constitution Art. I: no UI control, only observation).

**Why:** G1 in PRD requires visibility of 150+ sessions. UI is optional but useful observation layer. Must be read-only (Constitution non-negotiable).

---

## Acceptance Criteria

- [ ] Create `ui/` directory (separate from backend):
  - `src/main.tsx` (React entry point)
  - `src/App.tsx` (main layout)
  - `src/pages/Dashboard.tsx` (dashboard page)
  - `vite.config.ts`
  - `tailwind.config.ts` (styling)

- [ ] Layout structure:
  ```
  Dashboard
  ├── Sidebar (navigation)
  │   ├── Sessions
  │   ├── Agents
  │   ├── Tasks
  │   └── Hooks
  ├── Main Content Area
  │   ├── Header (timestamp, connection status)
  │   └── Active Section (Sessions | Agents | Tasks | Hooks)
  └── Footer (latency, message count)
  ```

- [ ] WebSocket integration:
  - Connect to `ws://127.0.0.1:8080/monitor?types=sessions,agents,tasks,hooks`
  - Show connection status (green = connected, red = disconnected)
  - Auto-reconnect on disconnect

- [ ] State management:
  - Use Zustand for client state (sessions, agents, tasks, hooks)
  - Update on WebSocket delta messages

- [ ] Read-only guarantee:
  - No buttons for state changes
  - No forms that modify data
  - Observation only (Constitution Art. I)

- [ ] Accessibility:
  - WCAG 2.1 AA minimum
  - Keyboard navigation
  - Screen reader friendly

- [ ] Tests: component rendering, WebSocket connection, state updates

---

## Implementation Notes

- **Constitution:** Art. I — UI is observation only, no control
- **Tailwind:** for rapid UI building, responsive design
- **Zustand:** lightweight state management (no Redux overhead)
- **Vite:** fast build, HMR in dev
