## Summary

Implement Hooks tab showing execution history of 19 active hooks. Display: hook name, last run, next run, status (success/failed), output snippet. Table with search + filter. Update via WebSocket deltas.

**Why:** G1 requires visibility. Hooks tab shows framework automation health, failures, next scheduled runs.

---

## Acceptance Criteria

- [ ] Create `ui/src/pages/Hooks.tsx`:
  - Table with columns: Hook Name, Last Run, Next Run, Status, Output, Duration
  - Sort: by name, by last run, by next run
  - Filters: show all, show failed only, show scheduled

- [ ] Columns:
  ```
  | Hook Name                | Last Run | Next Run | Status    | Duration |
  | UserPromptSubmit         | 2s ago   | —        | ✓ SUCCESS | 150ms    |
  | SessionStart             | 5m ago   | +1h 55m  | ✓ SUCCESS | 320ms    |
  | GithubPullRequest        | 1h ago   | 2026-06-04 (tomorrow) | ⚠ FAILED | 890ms |
  ```

- [ ] Interactivity (read-only):
  - Click row to expand → show full output (last 500 chars)
  - Expansion: hook code snippet, last 3 runs history
  - No edit/rerun buttons

- [ ] Status indicators:
  - ✓ SUCCESS (green)
  - ⚠ FAILED (red)
  - ⏱ RUNNING (blue, spinning)
  - ○ SCHEDULED (gray)

- [ ] Real-time updates:
  - Status changes animate
  - Last Run and Next Run update every 10s

- [ ] Search:
  - Filter by hook name
  - Search output by keyword

- [ ] Tests: table rendering, status display, filtering, real-time updates

---

## Implementation Notes

- **Output truncation:** only show first 500 chars (full output in expansion)
- **Duration:** milliseconds, helps spot slow hooks
- **Status animation:** brief pulse on status change
