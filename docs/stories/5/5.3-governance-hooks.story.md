# Story 5.3 — Deploy Pre-Commit Validation Hooks & Wire Test Scope

**Epic:** EPIC-5 (Framework Governance)  
**Status:** Ready

**Assignee:** @dev (Dex)

## Summary

Deploy 3 pre-commit validation hooks designed in EPIC-5 Phase 2 (SYNC-COMPLETE-REPORT §3) to catch governance violations before they reach the repository. Wire `tests/hooks/*` into the local `npm test` command to verify enforcement gates work in CI.

**Why:** The framework audit revealed drift patterns (V-DRIFT-004) and test gaps (V-ART5-005). Pre-commit hooks prevent the same issues from re-occurring; wiring tests ensures enforcement gates remain reliable.

**Source:** `docs/stories/epics/epic-5-governance/outputs/SYNC-COMPLETE-REPORT.md` (§3, §5 Immediate)

---

## Acceptance Criteria

- [ ] **AC1: Drift Guard Hook Deployed**
  - Hook: `.claude/hooks/pre-commit-drift-guard.cjs`
  - Function: Fails commit if a rule/hook references constitutional article not defined in `constitution.md`
  - Test: Create test commit with fake article reference → hook blocks
  - Lint: ESLint clean, no eval/SQL injection

- [ ] **AC2: Reference Hygiene Hook Deployed**
  - Hook: `.claude/hooks/pre-commit-reference-check.cjs`
  - Function: Fails commit if `.claude/rules/**/*.md` contains broken references (missing task, agent, file path)
  - Scope: Check `.aiox-core/data/`, `.claude/rules/`, `.claude/agents/`
  - Test: Create test reference to non-existent file → hook blocks

- [ ] **AC3: Constitutional Coherence Hook Deployed**
  - Hook: `.claude/hooks/pre-commit-constitution-check.cjs`
  - Function: Warns (non-blocking) if edits to `constitution.md` introduce new articles without adding them to gate enforcement files
  - Scope: Monitor `constitution.md`, `.claude/hooks/enforce-*.cjs`, `core-config.yaml`
  - Test: Edit constitution.md, run hook → produces warning log

- [ ] **AC4: Test Scope Wired**
  - Edit: `package.json scripts.test`
  - Change from: `"npm run jest"` (or current)
  - Change to: `"npm run jest && node --test tests/hooks/*.test.js"`
  - Test: Run `npm test` locally → all hook tests execute (verify 8/8+ tests run)
  - Lint: No breaking changes to existing jest suite

- [ ] **AC5: All Hooks Pass on Local Commit**
  - Pre-commit flow: Stage changes → run `git commit` → all 3 hooks execute without blocking
  - Success: Commit message appears in git log
  - Failure: Any hook block = story fails (investigate and fix before re-test)

- [ ] **AC6: Documentation Updated**
  - File: `docs/governance/pre-commit-hooks-guide.md` (new)
  - Content: Purpose, how to bypass (with audit trail), which violations each hook catches
  - File List: Updated in story

- [ ] **AC7: CodeRabbit Pre-Commit Code Review**
  - All 3 hook files reviewed for security (no eval, no command injection)
  - Verdict: PASS or CONCERNS (FAIL blocks story)

- [ ] **AC8: Gate Decision Logged**
  - Log file: `.aiox/gate-logs/story-5.3-hook-deployment-{date}.jsonl`
  - Entry: One decision per hook deployment, success/failure, timestamp
  - Compliance: Art. VI (Framework Boundary) — L1/L2 files not modified
