### OUT
- Criação de novas features (apenas remediação)
- Refactoring de código fora do escopo do framework
- Alterações a componentes L1 (protegidos por deny rules)

## Tasks / Subtasks

### FASE 1: Diagnóstico Completo (3 tasks paralelas)

- [ ] **Task 1.1: Baseline Audit** (Agent: @architect, ~90m)
  - [ ] Scan `.aiox-core/` completo
  - [ ] Scan `KAIROS_CEREBRO/` completo
  - [ ] Extrair metadados
  - [ ] Comparar estruturas vs AIOX reference
  - [ ] Identificar gaps e componentes órfãos
  - [ ] Output: `baseline-audit.json`

- [ ] **Task 1.2: Constitution Validation** (Agent: @qa, ~60m)
  - [ ] Validar Art. I (CLI First)
  - [ ] Validar Art. II (Agent Authority)
  - [ ] Validar Art. III (Story-Driven)
  - [ ] Validar Art. IV (No Invention)
  - [ ] Validar Art. V (Quality First)
  - [ ] Validar Art. VI-VII (Framework Boundary)
  - [ ] Detectar ambiguidades
  - [ ] Output: `constitution-violations.json`

- [ ] **Task 1.3: Failure Pattern Analysis** (Agent: @analyst, ~75m)
  - [ ] Ler `.aiox/gate-logs/`, `.aiox/metrics/`, `.synapse/metrics/`
  - [ ] Identificar padrões de falha recorrentes
  - [ ] Root cause analysis
  - [ ] Análise de frequência e impacto
  - [ ] Output: `failure-patterns.json`

- [ ] **Task 1.4: Data Integrity Check** (Agent: @data-engineer, ~60m)
  - [ ] Validar YAML parsing (zero erros)
  - [ ] Verificar referências task/workflow/agent
  - [ ] Detectar ciclos de dependência
  - [ ] Validar caminhos em rules files
  - [ ] Verificar frontmatter completo
  - [ ] Output: `data-integrity-report.json`

### FASE 2: Síntese & Remediação

- [ ] **Task 2.1: Synthesize Findings** (Agent: @aiox-master, ~45m)
  - [ ] Merge outputs de Task 1.1-1.4
  - [ ] Categorizar issues por severidade
  - [ ] Estimar effort para cada issue
  - [ ] Priorizar sequência e dependências
  - [ ] Identificar blockers
  - [ ] Gate: findings ≥5 issues AND solutions propostas
  - [ ] Output: `SYNC-FINDINGS.md` (8-12 páginas)

- [ ] **Task 2.2: Remediate Violations** (Agent: @dev, ~120m)
  - [ ] Fix Art. I — CLI features faltando
  - [ ] Fix Art. II — git push unauthorized
  - [ ] Fix Art. III — associar stories a código órfão
  - [ ] Fix Art. IV — rastreabilidade
  - [ ] Fix Art. V — lint/type/test/build failures
  - [ ] Fix Art. VI-VII — L1/L2 protection
  - [ ] Output: `remediation-commits.json`

- [ ] **Task 2.3: Fix Data Issues** (Agent: @data-engineer, ~60m)
  - [ ] Fix broken references
  - [ ] Fix YAML errors
  - [ ] Preencher frontmatter incompleto
  - [ ] Validar paths em rules
  - [ ] Output: `data-fixes.json`

- [ ] **Task 2.4: Clarify Ambiguities** (Agent: @architect, ~45m)
  - [ ] Document ambiguidades detectadas
  - [ ] Propor resoluções recomendadas
  - [ ] Implementar clarificações em rules
  - [ ] Output: `AMBIGUITIES-CLARIFIED.md`

- [ ] **Task 2.5: Process Improvements** (Agent: @architect, ~90m)
  - [ ] Criar validation hooks (pre-commit)
  - [ ] Criar governance health dashboard
  - [ ] Documentar SOPs
  - [ ] Criar runbooks
  - [ ] Output: `PROCESS-IMPROVEMENTS.md`

### FASE 4: Validação Final

- [ ] **Task 3.1: Final Validation** (Agent: @qa, ~30m)
  - [ ] Re-run baseline audit
  - [ ] Confirmar 0 violações activas
  - [ ] Confirmar 0 broken references
  - [ ] Confirmar 100% data integrity
  - [ ] Comparar before/after
  - [ ] Gate: violations=0 AND broken_refs=0 AND data_errors=0
  - [ ] Output: `SYNC-VALIDATION-FINAL.json`

- [ ] **Task 3.2: Final Report** (Agent: @architect, ~45m)
  - [ ] Executive summary (issues found, all resolved)
  - [ ] Findings & resolutions
  - [ ] Before/after measurements
  - [ ] Next steps (monthly sync, weekly health checks)
  - [ ] Appendices (audit results, commits, hooks, SOPs)
  - [ ] Output: `SYNC-COMPLETE-REPORT.md` (10-15 páginas)
