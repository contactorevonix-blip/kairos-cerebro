## Summary

Implement Sessions tab in dashboard showing real-time timeline of 150+ Claude Code sessions. Display: agent (@pm, @dev, etc.), story ID, token usage, status, duration. Sort by status (active first), then by start time. Update via WebSocket deltas every 500ms.

**Why:** G1 requires visibility into all sessions. Timeline shows what agents are doing, token burn, completion rate.

---

## Acceptance Criteria

- [ ] Create `ui/src/pages/Sessions.tsx`:
  - Table with columns: Agent, Story, Status, Started, Duration, Tokens (in/out), Prompts
  - Sort options: by agent, by status, by token usage
  - Filters: show active only, show paused, show done
  - Color coding: green=active, orange=paused, gray=done

- [ ] Columns:
  ```
  | Agent | Story | Status | Started | Duration | Tokens (in/out) | Prompts |
  | @dev  | 1.1   | active | 14:32   | 2m 15s   | 2.4K / 1.8K     | 5       |
  ```

- [ ] Interactivity (read-only):
  - Click row to expand → show full session details
  - Expansion shows: uuid, prompt_count, token_input, token_output, timestamps
  - No edit/delete buttons

- [ ] Real-time updates:
  - On WebSocket delta, update table immediately
  - Animate token changes (brief highlight)
  - Keep sort order consistent

- [ ] Search:
  - Filter by agent name (e.g., "@dev")
  - Filter by story (e.g., "1.1")

- [ ] Performance:
  - Render 150+ rows without lag (virtualize if needed)
  - Update <50ms per delta

- [ ] Tests: table rendering, sorting, filtering, real-time updates

---

## Implementation Notes

- **Virtualization:** use react-window for 150+ rows
- **Sorting:** client-side, no server query
- **Read-only:** no action buttons (Constitution Art. I)
- **Color coding:** helps visual scanning
