#### AC7 Resolution (the flagged ambiguity)

Read AC7, `research.json` RT-3, and the **canonical** `requirements.json` FR-4.2.
FR-4.2 is **not genuinely ambiguous**: its acceptance is explicitly *"Context size
> 937 lines"*, with rationale *"shim activation reads persona + tasks + workflows +
memory"* and flow INT-1 *"Shim loads (~102 lines) → Persona auto-loads →
Tasks/templates/memory load → Context reaches 937+ lines"*. The intent is the
**aggregate loaded context**, not the winning surface's own size.

The delivered reconciler resolves the **winning surface's raw content** and reports
*its* `lineCount`. Empirically (measured this review): Surface 1 `.claude/agents/aiox-dev.md`
= 102 ln (the shim, which wins per priority order); SKILL.md = 582 ln; template = 572 ln.
No single surface reaches 937 ln. The FR-4.2 unit test only proves the line counter
returns ≥937 when handed 950 synthetic lines — it never proves the real shim expands
to 937+. **AC7 is therefore genuinely unmet at the integration level** — but this is a
scope boundary (aggregate expansion belongs to pipeline enrichment / FR-4.1 Tier A
loading), **not a defect** in the well-engineered reconciler. → CONCERNS, routed to @po.
