## Story Statement

**As a** framework developer,  
**I want to** expose 8 commands that already exist as task files but are not yet wired in their agent definitions,  
**so that** the AIOX agent command surface matches the official documentation.

---

## Problem

Research (Cont 71, session 2026-06-24) confirmed that 8 commands are documented in the official `aiox-agent-flows/*-system.md` files but not exposed in the corresponding agent `commands:` sections in KAIROS_CEREBRO. The task files already exist (verified), so this is pure wiring work, not implementation.

**Examples:**
- `@dev *improve-code-quality` — task file exists (`dev-improve-code-quality.md`), not in dev.md `commands:`
- `@architect *collaborative-edit` — task file exists (`collaborative-edit.md`), not in architect.md
- `@devops *security-scan` — task file exists (`security-scan.md`), not in devops.md
