## Story

**As a** framework maintainer,
**I want** an `aiox metrics` CLI command that lists, retrieves, and exports metrics aggregated from existing log sources,
**so that** I can query framework health and history without manually grepping `.aiox/gate-logs/`, `.synapse/metrics/`, or `.aiox/task-logs/`.

---

## Acceptance Criteria

1. CLI command: `aiox metrics list` (show all available metrics)
2. CLI command: `aiox metrics get {metric-id}` (single metric + history)
3. CLI command: `aiox metrics export --format [json|csv|table]` (export for analysis)
4. Default data sources: gate-logs, hook-metrics, task-logs (no manual input)
5. Performance: <500ms for 30-day history query
6. Help text: `aiox metrics --help` complete + examples
