# Story 8.2.3: Impact Analysis Graph

**Status:** Ready
**Story ID:** 8.2.3
**Created:** 2026-06-11
**Epic:** EPIC-8 — Framework Evolution (Phase 2: IDS Enhancement)
**PRD Source:** `docs/prd/epic-8/phase-2-ids.md#823-impact-analysis-graph-2sp`
**Story Points:** 2

---

## Story

**As a** framework maintainer or PM planning a change,
**I want** to visualize REUSE→ADAPT→CREATE dependency chains as a queryable graph,
**so that** I can answer questions like "what breaks if I change entity X?" before making the change, instead of discovering blast radius after the fact.

---

## Acceptance Criteria

1. Data model: `.aiox-core/data/ids-impact-graph.json` (graph schema)
   - Nodes: Entity (task, template, script)
   - Edges: Depends-on, Reused-by, Adapted-from
2. Graph building: `aiox ids graph --format [json|dot|mermaid]`
3. Queries:
   - Show all entities reusing pattern X
   - Show adaptation chain (original → adapt1 → adapt2)
   - Detect circular adaptation chains
4. Visualization: ASCII + Mermaid diagram support
5. Performance: Build graph for 500 entities <5s

---

## Dependencies

## Risks

- **Risk:** AC5 performance target (<5s for 500 entities) is based on the PRD/research baseline of 474-500 entities; the live registry already has 823 entities.
- **Mitigation:** Benchmark against 823 entities. If the gap to <5s is large, document it as a known deviation rather than silently passing a weaker test — do not lower the AC without @po sign-off.

- **Risk:** "Adapted-from" edges (AC1, AC3) may not exist as a distinct relationship type in the current registry schema (`usedBy[]`/`dependencies[]` are observed; an explicit `adapted-from` edge type was not found in the registry sample).
- **Mitigation:** If `adapted-from` relationships are not present in `entity-registry.yaml`, derive them where possible from `metadata.created_justification` (introduced by Story 8.2.5) or document as a known gap for 8.2.5/8.2.6 to backfill. Do not invent fictitious adaptation chains.
