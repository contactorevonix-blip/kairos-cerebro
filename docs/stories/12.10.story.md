## Details

### Testing Approach

**MAESTRO Layer Validation:**
- **L1 (Core):** Design system framework, accessibility standards
- **L2 (Templates):** Component template, design token template
- **L3 (Config):** Design system config (colors, typography, spacing)
- **L4 (Runtime):** @ux operates on project components/design system

**Task-First Contract:**
```
Task: Test @ux-design-expert Agent
  Inputs: [feature requirement, accessibility requirements]
  Outputs: [component design, design token updates, accessibility audit]
  Success: All 8 ACs passing, WCAG 2.1 AA compliance verified
```

### UX Design Workflow

@ux follows design task:
1. **Feature Analysis:** Understand feature requirements (from story)
2. **Component Design:** Design component(s) needed, sketch spec
3. **Design System:** Apply/extend design tokens (color, typography, spacing)
4. **Accessibility:** Verify WCAG 2.1 AA compliance (keyboard nav, screen reader, contrast)
5. **Responsive Design:** Verify design works across breakpoints
6. **Handoff to Dev:** Pass component spec to @dev for implementation

### Research Alignment

**Clean Architecture** (research rec #1):
- Design is peripheral layer (business logic center)
- @ux owns design, @dev owns implementation

**Production Readiness** (research rec #5):
- Accessibility compliance required from start
- Design system maintains consistency
