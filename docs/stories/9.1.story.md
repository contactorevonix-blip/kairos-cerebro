# Story 9.1: Constitutional Gate Framework + Hook Integration

**Story ID:** 9.1  
**Epic:** EPIC-9 (Enforcement Gates)  
**Status:** Ready  
**Points:** 1sp  
**Owner:** @dev  
**Dependencies:** None (foundation for 9.2-9.5)

## Description

Implement base infrastructure for Constitutional enforcement gates at the PreToolUse hook layer. All 7 Constitutional articles (Art. I-VII) will hook into this framework to enforce policy deterministically.

## Scope

**IN:**
- Gate framework implementation (gate-runner)
- Hook integration to PreToolUse layer
- Decision logging (JSONL format, append-only)
- Metrics tracking infrastructure
- Graceful error handling (no hook crashes)
- 5+ test cases

**OUT:**
- Individual article gates (9.2-9.5 separate stories)
- UI/dashboard for gate visualization
- Alerting/notification system
- Performance optimization (baseline only)

## Acceptance Criteria

- [ ] Create `.claude/hooks/gate-framework.cjs` with gate-runner interface
- [ ] Gate-runner loads gate modules (enforce-agent-authority, enforce-story-driven, etc)
- [ ] Each gate decision logged to `.aiox/gate-logs/` as immutable JSONL (timestamp, article, decision, reason)
- [ ] Decision verdicts: `allow`, `block`, `override`, `warn`
- [ ] No gate can crash the hook (graceful error handling)
- [ ] Metrics: `gatesEnforced`, `violationsDetected`, `violationsBlocked`, `overridesUsed` tracked
- [ ] All 4 pre-commit gates PASS (lint ✅, typecheck ✅, test ✅, CodeRabbit pending)

## Risks Identified

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| Hook framework crashes on malformed gate config | Low | High | Validate gate configs at load time; graceful fallback |
| Decision logging performance degrades | Very Low | Medium | JSONL append is O(1); metrics in-memory until sync |
| Circular dependencies between gates | Low | High | DAG validation on framework initialization |
| Metrics file corruption on concurrent writes | Low | High | Use file locks (.lock) or atomic write patterns |
