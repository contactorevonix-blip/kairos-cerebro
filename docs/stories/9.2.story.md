# Story 9.2: Art. II Agent Authority Enforcement Gate

**Story ID:** 9.2  
**Epic:** EPIC-9 (Enforcement Gates)  
**Status:** InReview  
**Points:** 1sp  
**Owner:** @dev  
**Dependencies:** 9.1

## Description

Implement Constitutional Article II (Agent Authority) enforcement gate. Block @devops-exclusive operations (git push, PR create, release) when invoked by non-@devops agents. Gate hooks at PreToolUse layer.

**Article II Statement:** @devops has EXCLUSIVE authority over git push, gh pr create, and release operations. No other agent may perform these.

## Scope

**IN:**
- Detect @devops-exclusive operations (git push, gh pr create/merge, release)
- Read active agent from hook context
- Block if agent ≠ @devops (decision: block)
- Support `--skip-devops-check` override
- Decision logging to gate-logs
- 4+ test cases

**OUT:**
- Git workflow coaching (that's @devops responsibility)
- Release pipeline orchestration
- Agent authentication (that's @devops auth)
- Dashboard visualization of agent authority

## Risks Identified

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| False positive: @devops agent context not found | Medium | Medium | Try 4 sources (env, inline, session, fallback); test agent resolution |
| Override flag bypassed by malicious actor | Low | High | Audit trail in gate-logs; admins review overrides |
| Git command not recognized in hook input | Low | Low | Regex validation; test all git command variants |
| Performance: agent resolution on every git push | Very Low | Low | Cache agent context per session (~1ms lookup) |
