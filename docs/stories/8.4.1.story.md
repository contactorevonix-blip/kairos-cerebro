## Story

**As a** development team lead,
**I want** CodeRabbit auto-fix to handle more complex issues with increased iteration budget,
**so that** fewer CRITICAL/HIGH issues require manual intervention, reducing code review friction.

---

## Acceptance Criteria

- [x] **AC1:** Configuration `.claude/rules/coderabbit-integration.md` updated:
  - Dev phase: max_iterations increased from 2 → 3
  - QA phase: max_iterations increased from 3 → 5 (pre-merge)
  - MEDIUM issues: auto_fix enabled (was document_as_debt)
- [x] **AC2:** Circuit breaker implemented: If 3+ consecutive failed fixes → HALT, manual intervention required
- [x] **AC3:** All iterations logged to `.aiox/gate-logs/coderabbit-iterations.jsonl` (audit trail)
- [x] **AC4:** Performance: CodeRabbit runs in <5s per iteration (response cached where possible)
- [x] **AC5:** Safety: `--skip-coderabbit-auto-fix` flag allows developers to bypass auto-fix if needed
