# Story 13.1: Agent Definition Shim Consolidation

**Epic:** EPIC-13  
**Story ID:** 13.1  
**Status:** Ready  
**Effort:** 8 sp  
**Owner:** @dev (implementation) / @architect (design)  
**Priority:** 🔴 CRITICAL (Phase 1)

---

## Story Statement

**As a** framework developer,  
**I want** to consolidate agent definitions across L1/L2/L3 layers,  
**so that** changes in canonical sources propagate consistently to Claude Code without divergence.

---

## Problem

Agent definitions exist in **4 places**:
- `.aiox-core/development/agents/*.md` (L2 canonical source)
- `.claude/agents/*.md` (L3 legacy copies — may be stale)
- `.claude/commands/AIOX/agents/` (L3 shimmed + extras)
- `.claude/skills/AIOX/agents/*/SKILL.md` (L3 generated — actual runtime source for Claude Code)

**Risk:** Changes in L2 don't propagate to `.claude/skills/` or other targets. Agents can silently diverge.

---

## Acceptance Criteria

- [ ] **AC1:** Identify true source of truth: Is L2 the master, or are SKILLs generated/manually maintained?
- [ ] **AC2:** Verify all ideSync targets (`.codex/`, `.antigravity/`, `.claude/skills/`) are in sync with L2 source
- [ ] **AC3:** Remove dead `.claude/agents/*.md` copies or mark them clearly as deprecated
- [ ] **AC4:** Add pre-commit validation: prevent orphaned agent definitions (agents in L2 but missing from targets)
- [ ] **AC5:** Document the source-of-truth decision and ideSync dependencies in `.aiox-core/core/README.md`

---

## Scope

**IN:**
- Audit all 4 agent definition locations
- Trace dependencies: does generator script exist? If so, verify it runs in CI
- Identify dead code vs. intentional legacy
- Create pre-commit gate to detect drift

**OUT:**
- Rewriting agents themselves (only consolidation/audit)
- Modifying L1 core code
- Adding new agents

---

## Dependencies

None (read-only audit, write-only for docs/gates).

---

## Technical Context

### Data Points
- **L2 canonical:** `.aiox-core/development/agents/sm.md`, `dev.md`, `qa.md`, etc.
- **L3 SKILL files:** `.claude/skills/AIOX/agents/{name}/SKILL.md` — loaded at runtime by Claude Code
- **ideSync targets:** Defined in `core-config.yaml` under `ideSync` section
- **Pre-commit hooks:** `.claude/hooks/` directory

### Key Files to Audit
1. `.aiox-core/development/agents/` — list all
2. `.claude/agents/` — list all & check if they match L2
3. `.claude/skills/AIOX/agents/` — verify all L2 agents have SKILL counterparts
4. Check for a generator script (e.g., `generate-skills.js`)

### Validation Approach
- For each agent in L2:
  1. Check if SKILL.md exists in `.claude/skills/AIOX/agents/{name}/`
  2. If yes: is SKILL auto-generated (check header) or manually maintained?
  3. If no: document as missing target
- For each SKILL.md:
  1. Check if source exists in L2
  2. If no: is it intentional override? (document in comment)

## Tasks / Subtasks

1. **Audit L2 Agent Sources**
   - List all files in `.aiox-core/development/agents/`
   - Document count, names, update dates
   - Link: AC1, AC2

2. **Audit L3 Legacy Copies**
   - List all files in `.claude/agents/`
   - Compare filenames to L2 (which ones exist in L2? which don't?)
   - Document findings in decision log
   - Link: AC3

3. **Verify `.claude/skills/AIOX/agents/` Completeness**
   - For each L2 agent, check if SKILL.md exists
   - If missing, note as "missing target"
   - If present, check header for generation marker
   - Link: AC1, AC2

4. **Search for Agent Generator Script**
   - Search codebase for script that generates SKILLs from L2
   - Check if it runs in pre-commit or CI
   - If found, verify it's recent and working
   - If not found, mark sources as manually maintained
   - Link: AC1

5. **Document Decision: Single Source of Truth**
   - Write decision document: "Agent definitions will use [L2 / generated SKILLs / hybrid] as canonical source"
   - Document why (e.g., "SKILLs are generated from L2 in pre-commit")
   - Link: AC5

6. **Create/Update Pre-Commit Gate**
   - Design gate: "on pre-commit, verify all L2 agents have corresponding SKILL.md"
   - Implement as `.claude/hooks/validate-agent-sync.cjs`
   - Gate should report: missing SKILLs, orphaned SKILLs, outdated SKILLs
   - Link: AC4

7. **Update Core README**
   - Document agent source-of-truth location and ideSync flow
   - Include decision from Task 5 above
   - Link: AC5
