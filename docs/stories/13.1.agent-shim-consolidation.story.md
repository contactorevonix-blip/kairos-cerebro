## Story Statement

**As a** framework developer,  
**I want** to consolidate agent definitions across L1/L2/L3 layers,  
**so that** changes in canonical sources propagate consistently to Claude Code without divergence.

---

## Problem

Agent definitions exist in **4 places**:
- `.aiox-core/development/agents/*.md` (L2 canonical source)
- `.claude/agents/*.md` (L3 legacy copies — may be stale)
- `.claude/commands/AIOX/agents/` (L3 shimmed + extras)
- `.claude/skills/AIOX/agents/*/SKILL.md` (L3 generated — actual runtime source for Claude Code)

**Risk:** Changes in L2 don't propagate to `.claude/skills/` or other targets. Agents can silently diverge.

---

- [x] **AC1:** Identify true source of truth — DONE. L2 (`.aiox-core/development/agents/`) is canonical; SKILLs are **generated** (marker `ACORE-CLAUDE-AGENT-SKILL: generated`), via TWO pipelines: `ide-sync` (12 L2 agents) + `squad-creator` (46 experts). See decision-log-13.1.md §4-5.
- [x] **AC2:** Verify ideSync targets in sync — DONE. 12/12 L2 agents have SKILL counterparts (MISSING=0, ORPHANED=0, script-validated). Generator: `.aiox-core/infrastructure/scripts/ide-sync/index.js` writes 8 targets from `core-config.yaml ideSync:`. 44 squad SKILLs carry stale `Source:` headers (advisory, squad-managed).
- [x] **AC3:** `.claude/agents/*.md` — DONE (premise corrected). They are NOT dead copies: they are full **native Claude Code subagent definitions** (`tools:`, `permissionMode: bypassPermissions`), a distinct runtime layer. Documented as such; deletion would break subagent spawning. See decision-log §2.
- [~] **AC4:** Pre-commit drift gate — DESIGNED + logic VALIDATED against live repo. Implementation file `.claude/hooks/validate-agent-sync.cjs` is **L1-blocked** (settings.json deny `.claude/hooks/**`). Ready-to-apply source in decision-log §6 → apply via `@aiox-master *propose-modification`.
- [~] **AC5:** Document in `.aiox-core/core/README.md` — content WRITTEN (decision-log §7), but target is **L1-blocked** (settings.json deny `.aiox-core/core/**`). Ready-to-apply section provided → apply via `@aiox-master *propose-modification`.

### Validation Approach
- For each agent in L2:
  1. Check if SKILL.md exists in `.claude/skills/AIOX/agents/{name}/`
  2. If yes: is SKILL auto-generated (check header) or manually maintained?
  3. If no: document as missing target
- For each SKILL.md:
  1. Check if source exists in L2
  2. If no: is it intentional override? (document in comment)
