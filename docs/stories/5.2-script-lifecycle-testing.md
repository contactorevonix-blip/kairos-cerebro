# Story 5.2 — Script Lifecycle & Testing

**Epic:** AIOX-OPS (AIOX Operacional 100%) | **Story ID:** 5.2 | **Effort:** 6sp | **Status:** Ready | **Executor:** @dev | **Quality Gate:** @qa

## Story

**As a** AIOX automation system,
**I want** all hook scripts (`.js`/`.cjs`) to have automated tests and a version header,
**so that** we can verify lifecycle correctness and detect regressions — closing GAPs G2 and G5.

## Goal

Adicionar testes automáticos e cabeçalho de versão (`@version`, `@created`, `@modified`) a todos os scripts hook `.js` sem cobertura existente. Foco nos scripts da Story 1.20 que tinham "zero automated test coverage": `post-push-handoff-consolidate.js`, `update-state.js`, `state-sync.js`. Garantir que a lógica de rename/archive de handoffs (identificada como não-testada no STATE.md L46) está coberta.

## Contexto / Problema

GAP G2 (EPIC-AIOX-OPERACIONAL.md §3): STATE.md L46 regista "Zero automated test coverage on consolidation rename/archive logic despite Change Log claim". O hook `post-push-handoff-consolidate.js` não tem testes automáticos.

GAP G5 (EPIC-AIOX-OPERACIONAL.md §3): Scripts `post-story-update.js`, `update-state.js`, `state-sync.js`, `post-push-handoff-consolidate.js` sem cabeçalho de versão nem registo em manifesto. Sem versioning, não há rastreabilidade de mudanças.

## Acceptance Criteria

- [ ] **AC1:** Ficheiros alvo identificados e inventariados — os 4 scripts sem cobertura/versioning: `post-push-handoff-consolidate.js`, `update-state.js`, `state-sync.js`, `post-story-update.js`. Confirmar presença em `.claude/hooks/`.
  - *Given* o conjunto de scripts hook, *When* inspeccionados, *Then* cada ficheiro tem `file_exists: true/false` e `has_version_header: true/false`.

- [ ] **AC2:** Cabeçalho de versão adicionado a cada script alvo com os campos obrigatórios: `@version`, `@created`, `@modified`, `@description`, `@resolves` (story ID).
  - *Given* um script sem versioning, *When* o cabeçalho é adicionado, *Then* `grep "@version" <ficheiro>` retorna um valor semântico (ex: `1.0.0`).

- [ ] **AC3:** Testes automáticos criados para `post-push-handoff-consolidate.js` cobrindo a lógica de rename/archive. Mínimo: happy path (5+ handoffs → consolida), edge case (< 5 handoffs → não consolida), e preservação de conteúdo crítico (blockers, ADRs não são perdidos na consolidação).
  - *Given* 6 handoff YAMLs para o mesmo pipeline, *When* o hook é invocado, *Then* os 5 mais antigos são arquivados em `.aiox/handoffs/_archive/` e o mais recente permanece.

- [ ] **AC4:** Testes automáticos criados para `update-state.js` e `state-sync.js` cobrindo o comportamento de actualização de STATE.md — happy path e falha graceful (ficheiro inacessível).
  - *Given* STATE.md existe, *When* `update-state.js` é invocado com payload válido, *Then* STATE.md é actualizado sem corromper conteúdo existente.

- [ ] **AC5:** Todos os novos testes passam (`node --test tests/hooks/script-lifecycle.test.js`). Coverage report confirma que as funções alvo estão cobertas.
  - *Given* o test suite, *When* `node --test` corre, *Then* 0 failures, 0 errors.

- [ ] **AC6:** Manifesto de versioning actualizado (ou criado se não existir) em `docs/SCRIPT-VERSIONS.md` listando cada script com versão, data e story ID.

## Scope

**IN:**
- Scripts: `post-push-handoff-consolidate.js`, `update-state.js`, `state-sync.js`, `post-story-update.js`
- Adicionar cabeçalho de versão a cada um
- Criar testes automáticos (foco em consolidation rename/archive logic)
- Criar/actualizar manifesto `docs/SCRIPT-VERSIONS.md`

**OUT:**
- Scripts `.cjs` (enforcement hooks) — já testados em `tests/hooks/enforcement.test.js` (Story 1.20)
- Reescrever lógica existente dos scripts (apenas adicionar testes + headers)
- Modificar L1/L2 (`.aiox-core/core/`, `.aiox-core/development/`)
- Corrigir hooks partidos revelados por Story 5.1 (isso é scope das stories 5.4/5.5/5.6)

## Dependencies

- **Prerequisito:** Story 5.1 (Hook Automation Audit) — os outputs da auditoria confirmam quais scripts têm problemas adicionais que 5.2 deve evitar. 5.2 pode correr em paralelo desde que não dependa dos resultados de 5.1 para os scripts alvo específicos.

- **Recursos:** `.claude/hooks/*.js`, `tests/hooks/`, `docs/SCRIPT-VERSIONS.md` (a criar).

## Complexity Estimate

6sp — 4 scripts × (versioning header + testes) + novo test file. Média complexidade: a lógica de rename/archive tem edge cases (threshold 5+, preservação de campos críticos) que requerem testes cuidadosos.

## Business Value

Fecha a discrepância entre "Change Log afirma cobertura" e "cobertura real zero" identificada no STATE.md. Com testes automáticos, futuras mudanças nos scripts de handoff não passam silenciosamente — regressões são detectadas em CI/CD (`test.yml`).

## Risks

- **R1 (MEDIUM):** Lógica de `post-push-handoff-consolidate.js` pode ser mais complexa do que aparente — requer leitura cuidadosa antes de escrever testes. Mitigação: derivar test cases directamente do spec em `handoff-consolidation.md`.
- **R2 (LOW):** `update-state.js` e `state-sync.js` podem ter dependências de sistema que dificultam unit testing. Mitigação: usar mocks do `fs` module para isolar I/O.

### Regra de consolidação (fonte: `.claude/rules/handoff-consolidation.md`)

- Threshold: 5+ handoffs para o mesmo pipeline → MUST consolidate
- Preservar: blockers resolvidos, decisões arquitecturais, ficheiros modificados por wave
- Descartar: passos de investigação verbose, alternativas rejeitadas, sequences de tool calls
- Archive default: `.aiox/handoffs/_archive/{pipeline-slug}/`
