# Story 8.4.2: Self-Healing Story Validation

**Status:** InReview

## Story

**As a** product owner,
**I want** the story validation process to automatically detect and suggest fixes for vague AC, scope creep, or missing dependencies,
**so that** fewer stories get NO-GO verdicts and require re-submission cycles.

---

## Acceptance Criteria

- [x] **AC1:** Module `.aiox-core/core/auto-heal/story-validator.js` created with `validateStoryWithHealing(storyYaml)` function
- [x] **AC2:** Detection patterns implemented:
  - Vague AC (< 3 Given/When/Then clauses) → suggest structure
  - Scope creep (>10 AC items) → suggest split into sub-stories
  - Missing dependencies (references non-existent parent) → suggest link
  - Complexity mismatch (many AC but low effort) → suggest review
- [x] **AC3:** API returns: `{suggestions: [...], refined_story: {...}}`
- [x] **AC4:** Integration: `*validate-story-draft` command automatically runs healing and shows suggestions
- [x] **AC5:** @po reviews suggestions, approves/edits; reduces NO-GO verdicts by estimated 30%

---

## Dependencies

- **Depends on:** Story validation system active (from Phase 1-2)
- **Blocks:** None — enhances existing workflow

## Scope

**IN:**
- Story YAML parsing (AC, scope, description)
- Pattern matching on story structure
- Suggestion generation
- Refined story output

**OUT:**
- Automatic story fixing (user must approve)
- AI-based AC generation from description

---

## Risks

- **Risk:** Suggestions incorrect or inappropriate (false positives)
- **Mitigation:** @po can override; suggestions are advisory only
