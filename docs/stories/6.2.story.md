## Summary

Remove 2 dead hooks from `.claude/settings.json` that use incorrect protocol signature:
- `pre-tool-use-validator.cjs` (PreToolUse event, matcher `""`)
- `user-prompt-submit-validator.cjs` (UserPromptSubmit event, matcher `""`)

Both hooks export async functions but do NOT follow Claude Code hook protocol (stdin/stdout JSON communication via `node <file>`). They have no stdin-reading code, so they silently fail. Intended functionality (Art. II git push validation, Art. III story-driven validation) is already covered by working gates (`enforce-agent-authority.cjs`, `enforce-story-driven.cjs`).

**Impact:** MEDIUM — 5–10s timeout wasted per prompt, misleading configuration.

---

## Problem Statement

**Current State:**
- `.claude/settings.json` contains 2 dead hook entries

- Intended functionality already implemented in working gates
