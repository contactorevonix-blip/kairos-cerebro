## Story

**As a** framework governance officer,
**I want** the system to extract and validate exclusive operations from mentor agents,
**so that** cloned squads inherit proper authority constraints and cannot escalate privileges beyond the mentor.

---

## Acceptance Criteria

- [x] **AC1:** Module `squads/squad-creator/core/authority-matrix.js` created with `extractAuthorityMatrix(mentorAgent)` function
- [x] **AC2:** Extracts from mentor: exclusive operations (Article II gates, MUST NOT operations) — sourced from the delegation matrix in `.claude/rules/agent-authority.md` + `persona.exclusive_authority` note
- [x] **AC3:** Validation: Cloned squad cannot exceed mentor's authority (no privilege escalation) — `validateNoEscalation(mentorMatrix, cloneMatrix)`
- [x] **AC4:** Documentation generated at `.aiox/squad-dnas/{squad-id}-authority.md` listing inherited exclusive operations + constraints — `renderAuthorityDoc` + `saveAuthorityDoc`
- [x] **AC5:** Example: If @devops mentor, cloned squad-devops inherits "git push" exclusive, but cannot create global CI/CD rules — verified end-to-end against real `@devops` agent (5 exclusive ops incl. git push, PR create/merge, MCP, CI/CD, release)

## Scope

**IN:**
- Authority rule extraction from `.claude/rules/agent-authority.md`
- Operation classification (EXCLUSIVE, BLOCKED, ALLOWED)
- Inheritance validation

**OUT:**
- Dynamic authority adjustment (squad operators must request via escalation)
- Authority UI/dashboard
