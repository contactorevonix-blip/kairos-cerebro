**Epic:** EPIC-13  
**Story ID:** 13.5  
**Status:** Ready  
**Effort:** 5 sp  
**Owner:** @analyst (audit) / @sm (reclassification)

---

## Story Statement

**As a** product team,  
**I want** all story files organized into proper epic folders with no orphaned stories in the root,  
**so that** story structure is predictable and @sm can validate story existence reliably.

---

## Problem

`docs/stories/` root contains orphaned files:
- `WORKSPACE-CLEANUP.story.md` (not in any epic folder)
- `1.8-aiox-compliance-audit.md` (might be duplicate of `docs/stories/1/1.8.story.md`?)
- `1.14-memory-lifecycle-architecture.md` (might be duplicate of `1/1.14.story.md`?)

**Risk:** Duplicates cause confusion. Orphaned stories are forgotten or not tracked.

**Impact:** Story index (13.2) can't be complete. Story numbering is ambiguous.

---

- [ ] **AC2:** For each: determine if it's a duplicate, orphaned, or intentional

## Technical Context

### Root Files to Audit

Check `docs/stories/` for all `.md` or `.story.md` files **not in subdirectories**.

Expected findings:
- `WORKSPACE-CLEANUP.story.md` — unclear purpose, location?
- `1.8-aiox-compliance-audit.md` — duplicate of `docs/stories/1/1.8.story.md`?
- `1.14-memory-lifecycle-architecture.md` — duplicate of `1/1.14.story.md`?
- Others? (scan recursively)

### Reclassification Logic

For each orphaned file:

1. **Is it a duplicate?**
   - Check if same story exists in `docs/stories/{epic}/{epic}.{story}.story.md`
   - Compare file size, content (first 100 chars)
   - If yes: mark for deletion, keep epic-nested version

2. **Is it an orphaned story?**
   - Extract story number (e.g., `1.8` from `1.8-*.md`)
   - Find its epic folder (e.g., `docs/stories/epics/epic-1-...`)
   - If epic exists: move file there
   - If epic doesn't exist: create epic folder and move

3. **Is it deprecated/archive-worthy?**
   - Check content for phrases like "deprecated", "obsolete", "do not use"
   - Ask: is this story completed and archived, or still active?
   - If archived: move to `.archived/` with explanation

### Archive Folder Structure

```
docs/stories/.archived/
├── README.md (explanation of archive)
├── epic-1/
│   ├── 1.99-old-experiment.md (with archival note)
├── epic-X/
│   └── X.99-deprecated-story.md
```

## Tasks / Subtasks

1. **List All Root Files**
   - Run glob: `docs/stories/*.md` and `docs/stories/*.story.md`
   - Document count, filenames, paths
   - Link: AC1

2. **Analyze Each File**
   - For each root file:
     - Extract story number (if any)
     - Check if duplicate exists in epic folders
     - Check if orphaned
     - Check if deprecated
   - Create audit report
   - Link: AC1, AC2

3. **Identify Duplicates**
   - For suspected duplicates, compare:
     - File size, modification date
     - First 200 chars of content
     - Story title/ID
   - Decide: keep canonical (epic-nested), mark copy for deletion
   - Link: AC2, AC3

4. **Move Orphaned Stories**
   - For each orphaned story:
     - Determine correct epic folder
     - Move file to `docs/stories/epics/{epic-folder}/`
     - Update filename if needed (ensure consistency)
   - Link: AC4

5. **Create Archive Folder**
   - Create `docs/stories/.archived/` directory
   - Create `docs/stories/.archived/README.md` explaining archive purpose
   - Structure: organize by epic (subfolders) if many archived
   - Link: AC5

6. **Move Deprecated Stories**
   - For deprecated/obsolete stories:
     - Move to `.archived/{epic}/{story}.md`
     - Add archival note at top: "**ARCHIVED:** This story was completed/deprecated on YYYY-MM-DD. Reason: ..."
   - Link: AC5

7. **Update INDEX-AUTHORITATIVE.md**
   - Based on completed audit, update INDEX (from 13.2)
   - Remove orphaned root entries
   - Add archived entries with archival note
   - Verify all stories now have consistent schema
   - Link: AC6
