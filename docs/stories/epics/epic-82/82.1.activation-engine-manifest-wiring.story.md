### Post-Done defect SYN-001 — FR-3 manifest wiring regression (L4 hook)

**Discovered:** 2026-06-27 (post-Done, during Story 82.2 AC1/AC6 E2E re-validation)
**Severity:** Functional (silent) — manifest never reached the engine, so L3–L6 produced 0 rules.
**Location:** `.claude/hooks/synapse-engine.cjs` lines 64–65 (the FR-3 manifest wiring added by this story's Tasks 2–3).

**The bug (2 lines):**

```js
// BEFORE (buggy — introduced by 82.1 Task 3):
const manifestResult = parseManifest(runtime.cwd);   // passed the DIRECTORY, not the manifest file
manifest = manifestResult.manifest;                  // parseManifest returns {domains,...}; .manifest is undefined
```

`parseManifest(runtime.cwd)` reads a directory path → graceful-degradation branch → `{domains:{}}`,
and `.manifest` is `undefined`. Net effect: `engine.process(prompt, session, { manifest: undefined })`
→ `computeActiveLayers` saw an empty manifest and L3–L6 had nothing to resolve → **0 domains, 0 workflow/agent rules** despite L2/L3 being correctly activated by the session.

**The fix (2 lines):**

```js
// AFTER (correct):
const manifestResult = parseManifest(path.join(runtime.cwd, '.synapse', 'manifest'));
manifest = manifestResult;
```

`path` was already required at line 17 (used at line 62), so no new import. This is an L4 hook
(`.claude/hooks/`) — freely editable by @dev, no deny rule, no governance gate (unlike the L1
`l2-agent.js` blocker handled earlier in this story).

**Proof (real engine + real `.synapse/manifest`):**
- Manifest now parses **19 domains** incl. `WORKFLOW_STORY_DEV.workflowTrigger=story_development`, `.file=workflow-story-dev`.
- AC1 E2E: session `{active_agent:dev, active_workflow:story_development}` → L3 `workflow` layer status `ok`, **9 rules** (was 0).
- NFR-5: no-agent path still loads only L0/L1 (`constitution`, `global`).
- AC6: median pipeline **0.855ms** (N=31) << 100ms `PIPELINE_TIMEOUT_MS` cap.
- Regression: `tests/synapse/merge-precedence.test.js` 11/11, `tests/hooks/synapse-activation.test.js` 21/21.

**Why E2E tests didn't catch it earlier:** existing tests construct the manifest in-process and pass it
directly to `engine.process(...)` — they exercise the engine, not the *hook's* manifest-loading call.
The defect lived only in the hook's wiring of `parseManifest`, which no test invoked end-to-end.
