## Story

**As the** SYNAPSE engine,
**I want** the active agent/workflow/task written to the sessions file and the parsed manifest supplied to the layers,
**so that** L2 agent context is actually injected when an agent is active.

## Acceptance Criteria

1. **[G1-CLOSURE — TEST FIRST]** Given a `UserPromptSubmit` event with prompt `"@pm draft the epic"`, session_id S, and cwd, when the SYNAPSE hook chain runs (synapse-engine.cjs activation bridge), then `loadSession(S, .synapse/sessions)` returns a session with `active_agent.id === "pm"` (i.e. the agent id is present in the file the engine reads — `hook-runtime.js:56-60` — not only in `hook-metrics.json`), AND `SynapseEngine.process(prompt, thatSession, { manifest })` yields a result whose L2 layer metric is `status:"ok"` with `rules.length > 0` sourced from `.synapse/agent-pm`.
2. Given `active_agent.id == "pm"`, when the engine runs, then L2 injects the `.synapse/agent-pm` rules including `AGENT_PM_AUTH_*` lines.
3. Given a manifest with `AGENT_PM_AGENT_TRIGGER=pm`, when the engine builds layer context, then `manifest.domains.AGENT_PM.agentTrigger == "pm"` is available to L2 (`parseManifest` from `domain-loader.js:49` is invoked).
4. Given no domain-key match but `.synapse/agent-pm` exists, when L2 runs, then it falls back to loading that file directly (the dead-code path at `l2-agent.js:62-72` is reachable).
5. Given no agent in prompt and none in session, when the engine runs, then L0/L1 output is byte-identical to pre-story output (no regression).
6. `synapse-diagnostics` reports L2 status `active` with `rules > 0` for the active-agent case; 0 FAIL items.

### Activation Bridge Pattern (FR-1/FR-2/FR-3)

The existing QW-1 bracket-persist pattern (already in `synapse-engine.cjs:53-58`) calls `updateSession()`. The activation bridge extends this same block to also write `active_agent.id`. No new persistence mechanism — reuse the existing `updateSession()` call.

The key insight from the architect (PRD §3.1 + `implementation.yaml` `boundary_ruling`): calling `updateSession()` from L4 is consuming a public L1 API — it does NOT trip the `core-config.yaml:91-92` write boundary. The L1 amendment is only needed when modifying `.aiox-core/core/synapse/**` source files themselves.

### `parseManifest()` Wiring (FR-3)

`domain-loader.js:49` exports `parseManifest(cwd)` as a pure function. Call it in the hook:
```js
const { manifest } = parseManifest(cwd);
engine.process(prompt, session, { manifest });
```
`engine.js:264-265` already merges `processConfig` (which includes `{ manifest }`), and `engine.js:321-328` already forwards `manifest` into `buildLayerContext`. One-line call-site change in the hook — no L1 edit required for wiring.
