## Story

**As a** multi-prompt session,
**I want** active agent/workflow/task and history persisted with a 7-day TTL,
**so that** context survives across prompts and stale sessions are reclaimed automatically.

## Acceptance Criteria

1. Given `@pm` on prompt 1 and a plain follow-up prompt 2 (no agent signal), when prompt 2 runs, then `session.active_agent.id` is still `"pm"` (sticky — carried from the sessions file written in prompt 1).
2. Given a `*brief` command used on any prompt, then `session.history.star_commands_used` contains `"brief"` (deduplicated — no duplicates even if `*brief` appears on multiple prompts).
3. Given `session.history.domains_loaded_last`, then it reflects the layer domain files injected on the most recent prompt (updated on every prompt, not just the first).
4. Given a session file in `.synapse/sessions/` that is older than `staleTTLHours` (default 168h from `core-config.yaml:122-126`), when a new session's first prompt runs (TTL cleanup fires), then the stale file is removed from the directory.
5. Session writes are atomic and fire-and-forget: a write failure (disk full, permission error) MUST NOT block the prompt or throw out of the hook.

### Sticky Agent Logic (AC-1)

`activation_quality` field distinguishes `"explicit"` (agent named in prompt) vs `"sticky"` (carried from previous session). On a plain prompt:
- If `session.active_agent.id` is not null → keep it, set `activation_quality = "sticky"`
- If null → do nothing (no invention)
This logic lives in the activation bridge (already added in 82.1); 82.3 only needs to verify it persists correctly in the history.

### TTL Cleanup (AC-4)

`staleTTLHours` default = 168h (from `core-config.yaml:122-126`, field `immortality.sessionTTLHours`). The cleanup path in `hook-runtime.js:70-80` fires on certain conditions. Understand the trigger before writing tests — the test fixture may need to simulate the condition (e.g., a session created with a backdated `last_activity`).

### Single `prompt_count` Increment

82.1 writes the session at the start of the hook chain; 82.3 extends it at the end. Guard: do NOT call `prompt_count++` twice. Pattern: 82.1 writes `active_agent`; 82.3 appends `history.*`; only one of them calls `prompt_count++`. Coordinate via a flag or write order in `synapse-engine.cjs`.
