## Acceptance Criteria

1. **Integration test (positive):** Active `@pm` + `active_workflow.id = "story_development"` + `*brief` prompt → L0, L1, L2, L3, L7 inject; L4, L5, L6 are skipped — test passes.
2. **Integration test (no-agent regression):** No active agent in session, plain prompt → only L0 and L1 inject; no other layer active — test passes (byte-identical to pre-EPIC-82 output for L0/L1).
3. **Golden snapshot:** `<synapse-rules>` output is byte-stable for fixed inputs (same session state + same domain files + same prompt → identical output on repeated runs). Snapshot file is committed.
4. **Graceful-degradation matrix:** Removing each domain file in turn (`.synapse/agent-pm`, `.synapse/workflow-story-dev`, `.synapse/commands`) keeps the prompt succeeding and all other layers injecting their rules correctly. No layer removal cascades to a full pipeline failure.
5. **Performance gate (BLOCKING):** Median pipeline time < 100ms with all triggers firing (L0+L1+L2+L3+L7 active). This is an automated BLOCKING gate — test fails if median >= 100ms.
6. **Diagnostics gate:** `synapse-diagnostics` returns 0 FAIL items for a session with an active agent and a `*command` prompt. Runs via CLI as defined in 82.1.

### Golden Snapshot Strategy (AC3, NFR-4)

The snapshot captures the raw `<synapse-rules>` XML envelope for the fixed session + prompt `"*brief"` + the domain files frozen at the time of test authoring. Any change to domain file content or engine formatting will break the snapshot intentionally — this is a change-detector, not a constraint.

**When to update the snapshot:** after a deliberate change to engine output (e.g., adding a new mandatory L0 rule). Procedure: delete `tests/synapse/fixtures/golden-snapshot.txt`, run the test once to regenerate, commit the new snapshot with a clear commit message.

### Graceful-Degradation Matrix (AC4)

Test table (parameterised):

| Domain file removed | Expected behaviour |
|--------------------|--------------------|
| `.synapse/agent-pm` | L2 skipped; L0+L1+L3+L7 still inject |
| `.synapse/workflow-story-dev` | L3 skipped; L0+L1+L2+L7 still inject |
| `.synapse/commands` | L7 skipped; L0+L1+L2+L3 still inject |

For each case: rename the file to `.bak`, run the pipeline, restore. Assert (a) `process.exitCode !== 1`, (b) skipped layer has `rules === 0`, (c) remaining layers have `rules > 0`.
