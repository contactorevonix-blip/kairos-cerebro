## Story

**As the** SYNAPSE engine,
**I want** every layer validated and every failure isolated,
**so that** a bad or missing domain file never injects garbage and never blocks a prompt.

## Acceptance Criteria

1. Given a malformed or empty domain file for a layer (e.g. `.synapse/workflow-story-dev` with invalid content), when that layer runs, then the layer is skipped (logged to metrics as `errorLayer`) and all other active layers still inject their rules correctly.
2. Given rule text containing a `</synapse-rules>`-like sequence (or `<synapse-rules>` or `</system>`), when the formatter processes it, then the sequence is stripped or escaped so the `<synapse-rules>` envelope remains intact and cannot be prematurely closed.
3. Given a total payload size over the token budget (`getTokenBudget(bracket)`), when formatting, then the lowest-precedence layers are trimmed first; L0 (Constitution) is NEVER trimmed regardless of budget.
4. Given any layer throwing an unexpected exception during `_safeProcess`, when the engine runs, then the hook still emits a valid `UserPromptSubmit` payload (non-null, structurally correct) and does not re-throw.
5. The user prompt text is never echoed back inside injected `<synapse-rules>` output — a prompt containing unique sentinel text must NOT appear verbatim in the injected rules.

### Validation Model (FR-9)

Pre-injection check (per layer, after `layer._safeProcess` returns):
1. `result` is not null
2. `result.rules` is a non-empty array
3. At least one rule is a non-empty string

If check fails → skip layer, log `metrics.errorLayer[layerName] = { status: 'skipped', reason: 'empty_or_malformed' }`.

This is additive to the existing `_safeProcess` try/catch — the try/catch handles thrown exceptions; this check handles the "returns quietly with bad data" case.

### Error Handling Table (from PRD §3.4)

| Failure | Detection | Behaviour |
|---------|-----------|-----------|
| Missing domain file | `loadDomainFile` returns `[]` | Layer skipped, logged |
| Malformed manifest | `parseManifest` empty domains | L2/L3 skip via no-match; L1 fallback |
| Layer throws | `_safeProcess` try/catch | `metrics.errorLayer`, continue |
| Pipeline > 100ms | hrtime check at `engine.js:308-317` | Remaining layers skipped |
| Hook > 5s | `setTimeout` at `synapse-engine.cjs:92-98` | `process.exit(0)`, empty context |
| Session write fails | try/catch | Fire-and-forget, prompt unblocked |
| Validation fail (new) | rule list empty/malformed | Layer skipped, logged |

### Sanitization Rules (NFR-3)

Sequences to strip/escape from rule strings before injection:
- `</synapse-rules>` and case/whitespace variants
- `<synapse-rules>` (prevents nested envelope)
- `</system>` (prevents breaking system prompt boundary)
- Implement as a `sanitizeRule(str)` pure function in `formatter.js`; test independently

Per-rule max length: 2000 chars (configurable constant). Per-payload max: token budget from `getTokenBudget(bracket)`. Trim from lowest precedence up (L7 first → L6 → L5 → ... → L1); L0 immune.

### No Prompt Echo Guarantee (NFR-3, AC-5)

The user prompt is consumed in 82.1 for signal detection (agent/workflow/task) but MUST NOT be passed through to the formatter or appear in rule strings. Verify: no code path in `formatter.js` or the engine's rule-building logic accepts `prompt` as input. If the detection helpers in 82.1 have a `prompt` reference, confirm it is scoped to detection only (not stored in session or forwarded).
