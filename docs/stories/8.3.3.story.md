# Story 8.3.3: Squad Template Generation

**Status:** Done
**Story ID:** 8.3.3
**Created:** 2026-06-11
**Epic:** EPIC-8 — Framework Evolution (Phase 3: Squad Creator)
**PRD Source:** `docs/prd/epic-8/phase-3-squad-creator.md#833-squad-template-generation-2sp`
**Story Points:** 2

---

## Story

**As a** framework maintainer,
**I want** a template engine that generates complete `squad.yaml` from extracted mentor DNA + customization inputs,
**so that** new squads can be created with `aiox squad create --mentor {agent} --name {name} --focus {area}` in <10 minutes.

---

## Acceptance Criteria

- [x] **AC1:** Module `squads/squad-creator/core/squad-template-generator.js` created
- [x] **AC2:** Generated squad.yaml includes: agent definitions (voice/thinking DNA applied), squad metadata (squad_id, focus_area, created_from), inherited dependencies, inherited authority
- [x] **AC3:** CLI command `aiox squad create --mentor {agent} --name {name} --focus {focus}` creates `squads/{squad-id}/squad.yaml` (not git-committed, dev only) — handler shipped at `bin/commands/squad-create.js`; final `aiox squad` wiring is upstream (see Completion Notes — `bin/aiox.js` is L1, deny-listed at @dev layer)
- [x] **AC4:** Template engine uses `.aiox-core/development/templates/squad-tmpl.yaml` as base — that exact path does not exist in repo; engine tries it first, then falls back to the shipped `squad-template/squad.yaml`, then a bundled default (graceful degradation, see Completion Notes)
- [x] **AC5:** Generated squad includes audit trail: "Created from @{mentor} on {date}"

### Completion Notes List

- **IMPLEMENTED (YOLO mode). All 5 ACs met. 17 unit tests pass; project `npm run lint` + `npm run typecheck` pass; new files lint clean under `npx eslint`.**
- **IDS decision — CREATE (justified):** No existing JS module generates `squad.yaml` from mentor DNA. REUSED `parseAgentBlock` (voice-dna.js) for mentor loading and the `extractVoiceDNA`/`extractThinkingDNA` extractors (8.3.1/8.3.2) as DNA inputs. No third option existed to ADAPT.
- **AC4 path deviation (honesty):** The story specs `.aiox-core/development/templates/squad-tmpl.yaml`, which does NOT exist in the repo. The shipped template is `.aiox-core/development/templates/squad-template/squad.yaml`. `loadBaseTemplate()` tries the specced path first, then the shipped path, then a bundled default — never hard-fails (same graceful-degradation pattern as voice-dna `loadToneClasses`). If @po wants the literal specced filename to exist, that template file must be created upstream (L2, @dev cannot write `.aiox-core/development/templates/**` without it being an extend-only addition — deferred to avoid inventing a template).
- **AC3 CLI wiring deviation (boundary, honesty):** There is no `aiox` bin in this repo (`package.json` bin = `kairos` only); the framework `aiox` CLI is `bin/aiox.js`, which is **deny-listed** for @dev in `.claude/settings.json`. I shipped a standalone, dependency-free handler at `bin/commands/squad-create.js` (writable — only `bin/aiox.js`/`bin/aiox-init.js` are denied) exposing `run(argv)`. It is invokable directly (`node bin/commands/squad-create.js --mentor dev --name "X" --focus y`) and ready to be wired into `bin/aiox.js` with a one-line `require`. The final `aiox squad create` registration is an upstream change (route via `@aiox-master *propose-modification` or @devops) — @dev must not contour the L1 deny rule.
- **Dependency-free YAML (Art. I / no new deps):** Emits and re-parses squad.yaml via a bundled `toYaml`/`parseSimpleYaml`. `createSquad` round-trip-validates its own output (parse-after-emit) to satisfy the Risk mitigation ("validate YAML schema before saving"). Bug found+fixed during testing: quoted scalars containing `:` (e.g. `"tasks:a.md"`) were mis-split into maps by the list parser — fixed by detecting fully-quoted list items.
- **No invention (Art. IV):** Every generated field traces to the base template, the mentor DNA (itself derived from shipped agent YAML), or an explicit input (name/focus). Inherited authority falls back to an `inherited_from` marker when the mentor declares none — no fabricated permissions.
- **Gate note:** `tests/squad-creator/**` is still not in the `npm test` glob (package.json `test` script) — out of story scope to modify project config; tests run via `node --test tests/squad-creator/squad-template-generator.test.js`.
