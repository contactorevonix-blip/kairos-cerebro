## Summary

Design and implement SQLite schema for Monitor data durability. Tables: `sessions`, `agents`, `tasks`, `hooks`, `metrics`. Enable WAL mode for concurrent read/write. Implement migration system (Flyway-style SQL files) so schema can evolve without data loss.

**Why:** NFR1-5 require durable storage. SQLite WAL ensures ACID + concurrency. Migrations enable safe schema updates.

---

## Acceptance Criteria

- [ ] Create `src/db/schema/v001_initial.sql`:
  ```sql
  CREATE TABLE sessions (
    id TEXT PRIMARY KEY,
    agent TEXT NOT NULL,
    story TEXT,
    prompt_count INTEGER DEFAULT 0,
    token_input INTEGER DEFAULT 0,
    token_output INTEGER DEFAULT 0,
    status TEXT CHECK (status IN ('active', 'paused', 'done')),
    started_at TIMESTAMP NOT NULL,
    updated_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
  );
  
  CREATE TABLE metrics (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL,
    timestamp TIMESTAMP NOT NULL,
    token_input INTEGER,
    token_output INTEGER,
    FOREIGN KEY (session_id) REFERENCES sessions(id)
  );
  
  CREATE TABLE agents (
    id TEXT PRIMARY KEY,
    status TEXT CHECK (status IN ('idle', 'active')),
    last_activity TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
  );
  
  -- [similar for tasks, hooks]
  ```

- [ ] Enable WAL mode on init:
  ```sql
  PRAGMA journal_mode=WAL;
  PRAGMA synchronous=NORMAL;
  ```

- [ ] Create `src/db/migrations/` directory with migration files:
  - `001_initial.sql` (session setup above)
  - All migrations numbered and ordered

- [ ] Migration runner in `src/db/Migrator.ts`:
  - Reads `.db/migrations/` directory
  - Tracks applied migrations in `_migrations` table
  - Applies only pending migrations
  - Rolls back on failure (transaction)

- [ ] Create indices for performance:
  - `idx_sessions_status` on `sessions(status)`
  - `idx_metrics_timestamp` on `metrics(timestamp)`
  - `idx_agents_status` on `agents(status)`

- [ ] Tests: verify schema creation, migration ordering, WAL mode enabled
- [ ] Performance: initial table creation <100ms

## Implementation Notes

- **Database file:** `.aiox/monitor.db`
- **No ORM:** raw SQL for control + clarity (IDS principle)
- **WAL:** enables concurrent reads during writes
- **Migrations:** idempotent, replayable from schema.sql
