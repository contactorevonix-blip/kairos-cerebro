# Story 13.9: Performance Optimization + Cache

**Story ID:** 13.9  
**Epic:** EPIC-13 (Full Context Determinism)  
**Status:** Ready  
**Points:** 5sp  
**Owner:** @dev  
**Dependencies:** 13.1-13.8

## Description

Implement comprehensive caching and optimization strategy to ensure layer loading meets performance targets: <2s cold start, <500ms cached access, <15% token overhead. This includes LRU eviction, cache warming, and token budget tracking.

## Acceptance Criteria

- [ ] Cache strategy: LRU with configurable size limits per layer type
- [ ] Implement cache warming on first successful load (pre-populate cache)
- [ ] Performance: cold start <2s, cached <500ms (measured)
- [ ] Token budget: <15% overhead vs. baseline (measured and logged)
- [ ] Cache hit/miss ratio logged to `.aiox/context-load-logs/`
- [ ] Implement cache invalidation strategy (TTL-based, event-based on file change)
- [ ] Cache size monitoring and alerting (WARN if cache >100MB)
- [ ] All 4 pre-commit gates PASS
