Phase 3: Processing Maps

Story Development Cycle (5 Phases)

  • 1. Create (@sm) → story.md (Draft)
  • 2. Validate (@po) → 10-point gate + verdict
  • 3. Implement (@dev) → Code + File List
  • 4. QA Gate (@qa) → 7-point check: PASS/CONCERNS/FAIL
  • 5. Deploy (@devops) → git push to main

Quality Gates

@po's 10-Point: AC clarity, scope fit, AC count, dependencies, labels, single concern, estimation, DoD, risk, context

@qa's 7-Point: ACs passing, no regressions, code quality, performance, error handling, docs, security
