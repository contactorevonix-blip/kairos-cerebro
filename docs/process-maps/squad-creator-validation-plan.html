CRÍTICO
Voice DNA fraca (68/100) no único DNA existente
O alan-nicolas tem Thinking DNA 92 (excelente) mas Voice DNA 68 (fraco). Causa: documentação AIOX é institucional/co-autorada, não voz pessoal. Para Gene Kim, Rummler e outros — os livros publicados são fontes Tier 0 e Voice DNA deve ser mais forte.
→ Fix: usar livros publicados como fontes Tier 0 (The Phoenix Project, Improving Performance, etc.)
ALTO
pedro-valerio sem WebSearch/WebFetch
pedro-valerio só tem Read/Grep/Glob/Write/Edit. Não consegue verificar se fontes externas dos DNAs são legítimas. Só valida estrutura local dos ficheiros YAML. Isto é suficiente para validar veto conditions e fluxo, mas não para validar rastreabilidade de citações.
→ Fix para PM-7.2: pedro-valerio valida apenas estrutura/veto/fluxo. A verificação de citações fica com oalanicolas + squad-chief (QG-SC-5.1).
ALTO
Sem smoke tests documentados para oalanicolas e pedro-valerio
QG-SC-5.2 exige 3 smoke tests por agente. Não existe nenhum ficheiro de smoke tests para os 3 agentes do squad-creator. Só existe o mind_dna_complete.yaml do alan-nicolas como prova de trabalho.
→ Fix: definir 3 prompts de teste por agente antes de PM-7.2. Ver secção 5 deste documento.
MÉDIO
squad-chief tem bypassPermissions mas opera em contexto de segurança
bypassPermissions significa que pode escrever em qualquer ficheiro sem confirmação. Para criar squads em L4 isso é correcto. Mas deve ser monitorizado quando opera em wf-create-squad.yaml para garantir que não toca em L1/L2 por engano.
→ Fix: verificar deny rules em .claude/settings.json — estão activas para L1/L2, o que protege mesmo com bypass.
BAIXO
squad-chief referencia mind-research-loop.md mas o ficheiro não foi verificado
Phase 1 do wf-create-squad.yaml diz "Executar mind-research-loop.md (mínimo 3 iterações)". Precisamos confirmar que este ficheiro existe e tem o conteúdo correcto.
