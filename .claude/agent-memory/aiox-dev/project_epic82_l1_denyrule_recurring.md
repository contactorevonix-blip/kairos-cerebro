EPIC-82 (SYNAPSE Dynamic Injection) modifies the same L1 files — `.aiox-core/core/synapse/engine.js`, `output/formatter.js`, `layers/l2-agent.js`, `diagnostics/synapse-diagnostics.js` — across stories 82.1/82.2/82.4/82.6, all under the SINGLE umbrella amendment **ART-VII-2026-001** (APPROVED 2026-06-27, `.aiox/proposals/EPIC-82-L1-amendment.md`).

**Recurring blocker:** amendment approval ≠ deny-rule lift. `settings.json` deny `Write/Edit/MultiEdit(.aiox-core/core/**)` mechanically blocks @dev's tools regardless of amendment. @dev attempt returns "File is in a directory that is denied by your permission settings." Confirmed Story 82.1 (commit bf1b3b9 "lift deny rule" did NOT persist — deny still present at HEAD) and again Story 82.2 (2026-06-27).

**Established resolution (82.1 precedent):** "@devops lifted deny rule + applied fix" — @dev prepares exact ready-to-apply diffs in the Dev Agent Record, @devops (or @aiox-master) applies the L1 write. Art. II makes boundary enforcement @devops-exclusive. Do NOT spend cycles re-attempting the Edit; go straight to diff + handoff.

**THREE independent L1 enforcement layers (discovered 82.2, 2026-06-27):** (1) `settings.json` deny `Write/Edit/MultiEdit(.aiox-core/core/**)` — Edit-tool level; (2) `.claude/hooks/enforce-quality-gates.cjs` PreToolUse; (3) **husky git pre-commit** `.husky/pre-commit` → `scripts/validate-l1-l2-changes.js` — blocks ANY `git commit` staging files under `.aiox-core/core/` (also tasks/templates/checklists/workflows/infrastructure, bin/aiox*.js, constitution.md). Layer 3 is HARD-CODED with NO amendment-aware override and NO env flag — only `git commit --no-verify` bypasses it, and the gate message states "Only @devops or framework maintainers can modify these files." Lifting layer 1 (Edit deny) does NOT lift layer 3. So even when Pedro lifts the settings.json deny and the Edits land + validate, @dev still cannot COMMIT L1 — that is @devops authority. @dev must NOT use `--no-verify` (circumvents constitutional Art. II gate). 82.2 outcome: 8 L1 edits applied to disk + 11/11 tests PASS + lint clean + AC6 1.378ms, staged, but commit handed to @devops.

**How to apply:** For any future EPIC-82 L1 story, deliver L3/L4 parts directly, validate the algorithm via a scratchpad prototype (byte-identical functions), write the real test against the target engine exports (fails pre-diff with "is not a function"), then hand the diff to @devops. See [[story821-amendment-vs-denyrule-gap]].

**Path-scoped commit bypasses husky L1 gate WITHOUT --no-verify (SYN-001, 2026-06-27):** husky layer 3 (`validate-l1-l2-changes.js`) inspects `git diff --cached --name-only` — i.e. the staged index. But `git commit -F msg.txt -- <pathA> <pathB>` (explicit pathspec) makes git build a TEMPORARY index from HEAD + only those paths; the pre-commit hook's `git diff --cached` then sees ONLY the pathspec'd files. So an @dev L4-only commit (e.g. `.claude/hooks/synapse-engine.cjs` + story docs) passes the gate cleanly ("No L1/L2 framework files detected") EVEN WHILE `engine.js`/`formatter.js` sit staged in the real index — they're simply not part of the scoped commit and stay staged, untouched, for @devops. This is NOT a circumvention: the L1 files genuinely aren't in the commit. Gotcha: `-m`/`-F` must come BEFORE `--`, else the message is parsed as a pathspec. Used for commit 3bbf446.

**SYN-001 defect (post-Done on 82.1, fixed 2026-06-27):** L4 hook `.claude/hooks/synapse-engine.cjs` lines 64-65 (82.1 Task 3 regression) called `parseManifest(runtime.cwd)` (a DIRECTORY → graceful "missing file" branch → `{domains:{}}`) then unwrapped non-existent `.manifest` → engine got `manifest=undefined` → L3-L6 = 0 rules despite correct trigger activation. Fix (L4, no governance gate, @dev-committable): `parseManifest(path.join(runtime.cwd,'.synapse','manifest'))` + `manifest = manifestResult`. parseManifest returns `{domains,devmode,globalExclude}` DIRECTLY (no `.manifest` wrapper) and auto-sets each `domain.file = kebab(domainKey)` (e.g. WORKFLOW_STORY_DEV → `workflow-story-dev`). QA had mis-attributed this to the parser/FR-3; root cause was the hook's call site. Proven E2E: 19 domains, L3 workflow=9 rules, AC6 median 0.855ms. Lesson: in-process tests that pass a hand-built manifest to `engine.process()` do NOT exercise the hook's manifest-LOADING — TEST-001 (82.6) must assert L3 injection end-to-end through the hook path.
