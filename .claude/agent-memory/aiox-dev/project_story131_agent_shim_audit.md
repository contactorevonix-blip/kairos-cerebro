Story 13.1 (EPIC-13 Phase 1) — Agent Definition Shim Consolidation. Read-only audit
of agent definitions across layers.

**Findings (script-validated against live repo, 2026-06-21):**
- L2 canonical = 12 agents in `.aiox-core/development/agents/*.md`. This is the source
  of truth; `.aiox-core/core/doctor/checks/skills-count.js` treats this dir as authoritative.
- `.claude/skills/AIOX/agents/*/SKILL.md` = 58 generated SKILLs (marker
  `ACORE-CLAUDE-AGENT-SKILL: generated`). Split: 12 sourced from L2 (via `ide-sync`),
  2 valid `.claude/agents`-sourced (aiox-ux, squad), 44 stale-source squad experts
  (Source header points at non-existent `.claude/agents/<name>.md`; real source is
  `squads/<squad>/agents/<name>.md`, generated by squad-creator pipeline). MISSING=0, ORPHANED=0.
- Two generators: `ide-sync` (`.aiox-core/infrastructure/scripts/ide-sync/index.js`, story 6.19,
  config `core-config.yaml ideSync:`, 8 targets) for the 12 L2 agents; squad-creator for the 46.
- `.claude/agents/aiox-*.md` are NOT stale shims — they are full native Claude Code subagent
  definitions (`tools:`, `permissionMode: bypassPermissions`), a parallel runtime layer. Do NOT delete.

**Why:** Story flagged "agents can silently diverge"; audit confirms the native-subagent
layer is outside the ide-sync graph (the real divergence risk).
