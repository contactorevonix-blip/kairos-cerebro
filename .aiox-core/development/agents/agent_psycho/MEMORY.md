## Framework Cialdini — Aplicado ao KAIROS
```
1. RECIPROCIDADE
   → Free tier com 50 checks reais. Sem pedir nada primeiro.
   → "Experimenta. Sem cartão. Sem KYC."

2. COMPROMISSO
   → Activity feed mostra que outros estão a usar
   → Counter server-side: mesmo número em todos os devices

3. PROVA SOCIAL
   → "X verificações hoje" (counter real, não localStorage)
   → Testimonials reais acima do fold
   → Logos de tecnologias suportadas (Node, Python, PHP, Go...)

4. AUTORIDADE
   → Linguagem forense, não de marketing
   → Stats de fontes reais (MRC, LexisNexis, SEON)
   → "8 layers OSINT" → especificidade = credibilidade

5. SIMPATIA
   → Pedro é anónimo, mas KAIROS é o aliado
   → "Nós vs Eles": os burlões são o inimigo comum

6. ESCASSEZ
   → "Founder pricing" (urgência real, não artificial)
   → "Preço sobe conforme a rede cresce"
```

## Kahneman — Loss Aversion no KAIROS
```
Perder €2.000 dói 2x mais do que ganhar €2.000 alegra.

NUNCA: "Ganha segurança."
SEMPRE: "Não percas o que já construíste."

Enquadramento correcto:
→ "18% de fraude não detectada = €X de chargebacks/mês"
→ "1 chargeback = 2.5 meses de Starter plan"
→ "ROI calculado: €29 protege €890+ de risco médio"
```

## Sequência Emocional do Funil
```
1. VISITANTE chega → "Isto é para mim?" → mostrar ICP imediatamente
2. INTERESSE → "Funciona mesmo?" → prova social + stats reais
3. DESEJO → "Quanto custa o risco?" → urgency strip com ROI calc
4. ACÇÃO → "Começa agora" → CTA verde, zero fricção, free tier
5. RETENÇÃO → "Estou a ser protegido" → first check email + onboarding
```
